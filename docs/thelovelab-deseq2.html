DESeq2

Repositorio en GitHub | Autores: 466 estrellas · R

¿Qué es y para qué sirve?

DESeq2 es una herramienta de análisis estadístico para detectar diferencias en la expresión génica a partir de datos de secuenciación de ARN (RNA-seq). Utiliza un modelo de distribución binomial negativa para modelar la variabilidad biológica y técnica en los datos, proporcionando robustas estimaciones de expresión diferencial entre condiciones experimentales.

Instalación

DESeq2 se instala en R mediante el paquete BiocManager:

if (!require("BiocManager", quietly = TRUE))
    install.packages("BiocManager")

BiocManager::install("thelovelab/DESeq2")

Uso básico

Este ejemplo muestra el flujo básico de análisis. Se asume que se tiene una matriz de recuentos y un diseño experimental.

library(DESeq2)

# Crear objeto DESeqDataSet
dds <- DESeqDataSetFromMatrix(countData = counts,
                              colData = colData,
                              design = ~ condition)

# Realizar el análisis de expresión diferencial
dds <- DESeq(dds)

# Extraer resultados
res <- results(dds)

# Ver los genes más diferencialmente expresados
head(res, 10)

Flujo de trabajo típico

El flujo habitual de análisis con DESeq2 incluye:

  1. Obtener recuentos de genes desde alineamientos (por ejemplo, con HTSeq o featureCounts).
  2. Preparar un diseño experimental (factor de condiciones).
  3. Construir un objeto DESeqDataSet.
  4. Calibrar el modelo con DESeq().
  5. Extraer resultados con results().
  6. Visualizar resultados (por ejemplo, con plotMA() o heatmap).

Errores comunes y buenas prácticas

  • Errores comunes:
    • No incluir la variable de diseño correcta en el modelo.
    • Usar datos de recuento sin normalización adecuada (DESeq2 realiza su propia normalización interna, pero la entrada debe ser recuentos brutos).
    • Ignorar la detección de outliers en el diseño experimental.
  • Buenas prácticas:
    • Verificar la calidad de los datos de entrada antes del análisis.
    • Usar plotMA() o plotPCA() para visualizar resultados previamente.
    • Aplicar corrección de múltiples pruebas (por ejemplo, ajuste de FDR con padj).
