¿Qué es y para qué sirve?

Este repositorio contiene flujos de trabajo (workflows) escritos en WDL (Workflow Description Language) para el procesamiento de datos de secuenciación de alta capacidad con GATK4 y herramientas relacionadas. Está diseñado principalmente para el descubrimiento de variantes genéticas, siguiendo un flujo estándar de análisis en bioinformática. Los workflows incluyen pasos clave como la alineación de reads, corrección de calidad, marcado de duplicados, recalibración de calidad de base y llamado de variantes.

Instalación

Este es un repositorio de flujos de trabajo, no un paquete instalable como una librería. Para usar estos workflows, necesitas:

  • Un entorno de ejecución compatible con WDL (como cromwell).
  • La herramienta gatk instalada y disponible en el PATH.
  • El repositorio clonado localmente o descargado.

Instalación de Cromwell (ejemplo con Docker): 

docker pull broadinstitute/cromwell:latest

Instalación de GATK4: 

conda install -c bioconda gatk=4

Clonar el repositorio: 

git clone https://github.com/gatk-workflows/gatk4-data-processing.git

Uso básico

Para ejecutar un workflow, necesitas un archivo de entrada en formato JSON y ejecutar cromwell con el workflow WDL y el archivo de entrada. 

docker run --rm -v $(pwd):/wd -w /wd broadinstitute/cromwell:latest java -jar cromwell.jar run gatk4-data-processing.wdl -i inputs.json

Ejemplo de inputs.json (ilustrativo; verifica los campos exactos en cada workflow): 

{
  "gatk4_data_processing.gatk4_input": "path/to/your/reads.fastq.gz",
  "gatk4_data_processing.reference_genome": "path/to/hg38.fa",
  "gatk4_data_processing.output_prefix": "output_prefix"
}

Flujo de trabajo típico

El flujo típico sigue estos pasos:

  1. Alineamiento de reads con herramientas como bowtie2 o hisat2.
  2. Marcar duplicados con GATK MarkDuplicates.
  3. Recalibrar calidad de base con GATK BaseRecalibrator y ApplyBQSR.
  4. Validar el alineamiento con GATK ValidateSamFile.
  5. Llamar variantes con GATK HaplotypeCaller.
  6. Realce de variantes (si aplica).

Errores comunes y buenas prácticas

  • Errores de ruta: Asegúrate de que los archivos de entrada (reads, referencia) estén en rutas accesibles y que los permisos sean correctos.
  • Problemas con el archivo de entrada: Verifica que el archivo inputs.json tenga los nombres de los parámetros exactos y los valores correctos. Usa el comando cromwell con la opción --inputs para validar.
  • Memoria insuficiente: Los workflows pueden consumir mucha memoria. Asegúrate de asignar suficiente RAM al contenedor Docker o al entorno de ejecución.
  • Buenas prácticas:
    • Usa referencias genómicas preprocesadas (índices) para mejorar el rendimiento.
    • Siempre valida el alineamiento y los archivos SAM/BAM con GATK ValidateSamFile.
    • Documenta tus flujos y versiones de herramientas para reproducibilidad.
