trimmomatic

https://github.com/timflutre/trimmomatic · 152 estrellas · Java

¿Qué es y para qué sirve?

Trimmomatic es una herramienta de corte de lecturas para datos de secuenciación de próxima generación (NGS) de Illumina. Está diseñada para eliminar bases de baja calidad, secuencias de adaptadores y otras contaminaciones de los datos de secuenciación, mejorando así la calidad de los datos de entrada para análisis posteriores como alineamiento, ensamblaje o variantes.

Instalación

Trimmomatic está escrito en Java, por lo que requiere un entorno Java 8 o posterior. Puedes instalarlo directamente desde el repositorio oficial usando el siguiente comando:

java -jar /ruta/a/trimmomatic-0.39.jar

El archivo JAR se puede descargar desde la página de releases del repositorio GitHub: https://github.com/timflutre/trimmomatic/releases. Asegúrate de usar una versión compatible con tu sistema y datos.

Uso básico

Un ejemplo de uso típico de Trimmomatic para cortar lecturas paired-end (PE) con eliminación de adaptadores y filtros de calidad:

java -jar trimmomatic-0.39.jar PE -phred33 \
  input_R1.fastq.gz input_R2.fastq.gz \
  output_R1_paired.fastq.gz output_R1_unpaired.fastq.gz \
  output_R2_paired.fastq.gz output_R2_unpaired.fastq.gz \
  ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \ 
  SLIDINGWINDOW:4:15 \ 
  MINLEN:36

Este comando realiza tres pasos:

  • ILLUMINACLIP: elimina adaptadores de Illumina usando el archivo de referencia TruSeq3-PE.fa.
  • SLIDINGWINDOW: aplica una ventana deslizante de tamaño 4 para eliminar bases con calidad promedio inferior a 15.
  • MINLEN: filtra lecturas que quedan con menos de 36 bases.

Nota: La sintaxis exacta puede variar según la versión. Siempre verifica la documentación oficial del proyecto.

Errores comunes y buenas prácticas

  • Usar el archivo de adaptadores incorrecto: Asegúrate de seleccionar el archivo de adaptadores correcto (por ejemplo, TruSeq3-PE.fa para secuenciación TruSeq PE).
  • Establecer umbrales demasiado estrictos: Filtros muy severos (como MINLEN:50) pueden eliminar demasiadas lecturas. Prueba valores razonables basados en tu tipo de secuenciación y longitud de lectura.
  • No especificar el formato de calidad: Usa -phred33 para datos de Illumina con calidad ASCII 33 (común en datos más antiguos) o -phred64 para datos con calidad ASCII 64. El error más común es usar el formato equivocado.
  • Ignorar el análisis de calidad previo: Antes de usar Trimmomatic, verifica la calidad de tus datos con herramientas como FastQC para entender el tipo de contaminación.
  • Reutilizar archivos JAR sin verificar la versión: Siempre descarga la última versión disponible del repositorio para asegurarte de tener correcciones y mejoras.
