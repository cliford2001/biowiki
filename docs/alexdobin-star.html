¿Qué es y para qué sirve?

STAR (Spliced Transcripts Alignment to a Reference) es un alineador de secuencias de RNA-seq de alta eficiencia y precisión, diseñado para alinear lecturas de RNA de alta profundidad a genomas de referencia, incluso en presencia de splicing alternativo. Es ampliamente utilizado en estudios de transcriptómica para detectar isoformas, identificar eventos de splicing y cuantificar expresión génica.

Instalación

STAR se distribuye como código fuente. Para su compilación, se requiere un compilador C++ compatible (como g++). 

git clone https://github.com/alexdobin/STAR.git
cd STAR
make

Tras compilar, el ejecutable STAR se generará en el directorio raíz. Puedes copiarlo a un directorio del PATH para usarlo globalmente: 

sudo cp STAR /usr/local/bin/

Uso básico

Un ejemplo de uso típico consiste en alinear lecturas de RNA-seq (en formato FASTQ) a un genoma de referencia previamente indexado. Este comando es ilustrativo; el uso real puede requerir ajustes según el tipo de datos. 

STAR \
  --genomeDir /path/to/genome/index \
  --readFilesIn reads_1.fastq reads_2.fastq \
  --outFileNamePrefix output/ \
  --outSAMtype BAM SortedByCoordinate

Nota: Los parámetros exactos pueden variar según la versión de STAR y el tipo de datos. Consulta la documentación oficial para opciones detalladas.

Flujo de trabajo típico

El flujo habitual con STAR incluye:

  1. Generar un índice del genoma de referencia usando STAR --runMode genomeGenerate.
  2. Alinar lecturas de RNA-seq a dicho índice.
  3. Post-procesar los resultados (por ejemplo, cuantificar genes con tools como HTSeq o featureCounts).

Errores comunes y buenas prácticas

  • Índice incompleto: Asegúrate de que el directorio genomeDir contiene todos los archivos generados por STAR --runMode genomeGenerate.
  • Formato de entrada: Verifica que los archivos FASTQ estén en formato correcto (no comprimidos, a menos que se especifique).
  • Memoria: STAR requiere mucha memoria para grandes genomas o profundidades altas. Ajusta --outSAMattributes o usa --limitOutSAMoneReadBufferN para controlar el uso.
  • Parámetros de splicing: Para datos de splicing complejo, considera ajustar --sjdbOverhang (usualmente 99 para RNA-seq).
  • Verificación: Siempre valida los resultados con herramientas de control de calidad como samtools flagstat o Qualimap.
