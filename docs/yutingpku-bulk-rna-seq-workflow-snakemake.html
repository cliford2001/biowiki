Bulk_RNA-Seq_Workflow_Snakemake

Repositorio · 14 estrellas · Python

¿Qué es y para qué sirve?

Bulk_RNA-Seq_Workflow_Snakemake es un flujo de análisis completo para datos de secuenciación masiva de ARN (bulk RNA-seq) desarrollado en Python utilizando Snakemake como sistema de gestión de pipelines. Automatiza todo el proceso desde el alineamiento de lecturas hasta el análisis diferencial de expresión y la anotación funcional, integrando herramientas clave como STAR, Cufflinks, DESeq2 y GO analysis. Ideal para investigadores que buscan un enfoque estandarizado, reproducible y escalable en el análisis de transcriptómica.

Instalación

Este flujo de trabajo requiere Python y Snakemake. Instala el entorno usando conda:

conda env create -f environment.yml
conda activate bulk-rna-seq-workflow

Alternativamente, si no tienes conda, instala Snakemake con pip:

pip install snakemake

Uso básico

Ejecuta el flujo de trabajo desde el directorio raíz con:

snakemake --profile profiles/slurm

Nota: El perfil slurm requiere una configuración previa en profiles/slurm. Asegúrate de que config.yaml contenga los parámetros correctos (campos de entrada, rutas, etc.) antes de ejecutar.

Flujo de trabajo típico

El flujo de trabajo típico sigue estos pasos:

  1. Alineamiento de lecturas con STAR.
  2. Generación de archivos de expresión con Cufflinks.
  3. Identificación de genes diferencialmente expresados usando DESeq2.
  4. Análisis funcional mediante GO analysis.

Errores comunes y buenas prácticas

  • Verifica los paths en config.yaml: Asegúrate de que las rutas a los datos y a las herramientas sean correctas y accesibles.
  • Usa perfiles adecuados para el entorno: Si trabajas en un cluster, configura el perfil slurm o lsf correctamente. El uso incorrecto puede causar fallos de ejecución.
  • Revisa la versión de las herramientas: DESeq2 y Cufflinks tienen versiones específicas que pueden no ser compatibles. Usa la versión especificada en environment.yml o requirements.txt.
  • Verifica el formato de los datos de entrada: Los archivos de secuenciación deben estar en formato .fastq o .fq, y las muestras deben estar bien etiquetadas en el archivo de diseño.
  • Procesa grandes conjuntos con paralelización: Aprovecha el sistema Snakemake para ejecutar tareas en paralelo. Asegúrate de que el entorno de ejecución (como SLURM) tenga suficiente memoria y tiempo asignado.
