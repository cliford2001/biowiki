Scanpy

Repositorio · 2507 estrellas · Python

¿Qué es y para qué sirve?

Scanpy es una herramienta de análisis de datos de secuenciación de células individuales (single-cell RNA-seq) desarrollada en Python. Está diseñada para manejar grandes conjuntos de datos, escalando a más de 100 millones de células. Permite realizar análisis integrados, incluyendo preprocesamiento, análisis de expresión génica, visualización, clustering y análisis de diferenciación celular, todo dentro de un entorno de código Python eficiente y extensible.

Instalación

pip install scanpy

Para un entorno de análisis de datos biológicos, se recomienda usar conda:

conda install -c conda-forge scanpy

Uso básico

A continuación se muestra un ejemplo ilustrativo de uso mínimo y realista para cargar datos de expresión génica y realizar un análisis básico:

import scanpy as sc

# Cargar datos (por ejemplo, desde un archivo .h5ad)
adata = sc.read_h5ad("datos_celula.h5ad")

# Preprocesamiento
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)

# Selección de genes variables
sc.pp.highly_variable_genes(adata, n_top_genes=2000)

# Reducción dimensional
sc.tl.pca(adata, n_comps=50)
sc.pp.neighbors(adata)
sc.tl.umap(adata)

# Clustering
sc.tl.leiden(adata)

# Visualización
sc.pl.umap(adata, color=["leiden", "gene_name"], legend_loc="on data")

Nota: La sintaxis ilustrativa; verifica en la documentación oficial del proyecto para los parámetros exactos y métodos disponibles.

Flujo de trabajo típico

Errores comunes y buenas prácticas

  • Uso de memoria: Scanpy puede consumir mucha memoria con grandes conjuntos de datos. Usa adata = adata[:50000] para pruebas o trabaja con subconjuntos si es necesario.
  • Preprocesamiento inadecuado: Asegúrate de normalizar correctamente los datos antes de reducir la dimensionalidad. No saltes sc.pp.normalize_total() y sc.pp.log1p() si no tienes datos ya procesados.
  • Seleccionar genes variables: Ajusta el número de genes variables según tu conjunto de datos (por ejemplo, 2000-3000 para datasets típicos).
  • Visualización: No uses sc.pl.umap() con muchos clusters sin etiquetar; usa legend_loc="on data" o legend_fontsize para mejorar la legibilidad.
  • Versiones: Mantén actualizada la versión de Scanpy y sus dependencias (AnnData, numpy, scipy).
