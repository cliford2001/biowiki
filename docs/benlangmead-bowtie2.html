Bowtie2

Repositorio en GitHub · 801 estrellas · C++

¿Qué es y para qué sirve?

Bowtie2 es un alineador de lecturas de secuenciación de nueva generación (NGS) rápido y sensible, diseñado para alinear lecturas de ADN o ARN a un genoma de referencia, incluso cuando existen inserciones o deleciones (alineamiento con huecos). Es particularmente útil en estudios de secuenciación de whole genome, whole exome o transcriptómica, donde se requiere alta sensibilidad para detectar variantses y estructuras complejas.

Instalación

Bowtie2 no está disponible en PyPI o CRAN, ya que está escrito en C++. Se distribuye como binario precompilado o se puede compilar desde el código fuente.

# Descargar el binario desde GitHub (recomendado para usuarios que no compilan)
wget https://github.com/BenLangmead/bowtie2/releases/latest/download/bowtie2-2.5.4-linux-x86_64.zip
unzip bowtie2-2.5.4-linux-x86_64.zip

# Mover el binario al directorio de ejecutables
sudo mv bowtie2-2.5.4/bowtie2* /usr/local/bin/

# Verificar la instalación
bowtie2 --version

Uso básico

El uso más común es alinear lecturas de un archivo FASTQ a un índice de referencia previamente construido. A continuación se muestra un ejemplo ilustrativo de un flujo típico:

# Paso 1: Construir el índice de referencia (solo una vez)
bowtie2-build reference.fasta reference_index

# Paso 2: Alinear lecturas
bowtie2 -x reference_index -1 reads_1.fastq -2 reads_2.fastq -S alignment.sam

Nota: La sintaxis exacta puede variar según la versión. Verifica en la documentación oficial.

Flujo de trabajo típico

El flujo típico de análisis con Bowtie2 sigue estos pasos:

  1. Preparar el genoma de referencia (FASTA).
  2. Construir el índice con bowtie2-build.
  3. Alinear lecturas (pares o simples) con bowtie2.
  4. Convertir el resultado SAM a BAM, ordenarlo y posicionar (usando herramientas como Samtools).
  5. Realizar análisis adicionales (llamado de variantes, expresión, etc.).

Errores comunes y buenas prácticas

  • Índice no construido: Asegúrate de ejecutar bowtie2-build antes de alinear. El índice se crea una vez y se reutiliza.
  • Formato incorrecto de entrada: Verifica que las lecturas estén en formato FASTQ o FASTA, y que los archivos estén correctamente nombrados.
  • Memoria insuficiente: Bowtie2 puede consumir mucha RAM si el genoma es grande. Usa índices reducidos o ajusta los parámetros de memoria.
  • Parámetros de alineamiento: Ajusta --very-sensitive, --very-fast, o --no-mixed según el balance entre sensibilidad y velocidad que necesites.
  • Salida en formato SAM: Usa -S para salida SAM. Para BAM, usa -S - y redirige a Samtools para convertir.
