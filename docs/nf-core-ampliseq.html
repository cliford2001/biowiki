nf-core/ampliseq

Repositorio en GitHub · 253 estrellas · Nextflow

¿Qué es y para qué sirve?

nf-core/ampliseq es un flujo de trabajo automatizado para el análisis de secuenciación de amplicones (amplicon sequencing), diseñado para ser robusto, reproducible y fácil de usar. Utiliza QIIME2 y DADA2 para realizar la calidad de lecturas, la denoising, la asignación de OTUs o ASVs, y el análisis taxonómico. Es ideal para estudios microbiológicos, como la caracterización de comunidades microbianas en muestras de suelo, agua, intestino, entre otros.

Instalación

Este flujo de trabajo se ejecuta con Nextflow y requiere QIIME2 (versión 2023.5 o superior) instalado. Sigue estos pasos: 

conda create -n qiime2-2023.5 qiime2=2023.5
conda activate qiime2-2023.5

Instala Nextflow:

curl -fsSL get.nextflow.io | bash
sudo mv nextflow /usr/local/bin

Uso básico

Ejecuta el flujo de trabajo con un archivo de configuración y datos de entrada: 

nextflow run nf-core/ampliseq \ 
  -profile docker \ 
  --input samplesheet.csv \ 
  --outdir results \ 
  --email your.email@example.com

Nota: El ejemplo anterior es ilustrativo. La sintaxis exacta de los parámetros debe verificarse en la documentación oficial del flujo.

Flujo de trabajo típico

El flujo de trabajo típico sigue estos pasos:

  1. Recopilación de datos: secuencias en formato FASTQ (emparejadas o no).
  2. Calidad: filtrado de calidad y truncado de lecturas.
  3. Denoising: uso de DADA2 para generar ASVs (Unidades de Secuencias Amplicon).
  4. Clasificación taxonómica: asignación de identidad a cada ASV.
  5. Generación de tablas: frecuencia de ASVs por muestra.
  6. Visualización: análisis y representación gráfica de resultados.

Errores comunes y buenas prácticas

  • Problemas con el entorno QIIME2: Asegúrate de activar el entorno correcto antes de ejecutar el flujo.
  • Archivos de muestra mal formateados: El archivo samplesheet.csv debe tener las columnas correctas: sample_name, forward, reverse.
  • Uso de perfiles: Usa --profile docker para evitar conflictos de dependencias. También puedes usar --profile singularity si tienes Singularity instalado.
  • Verificación de resultados: Revisa los archivos de log y los informes generados para detectar errores en el pipeline.
  • Documentación oficial: Siempre consulta la documentación del proyecto en GitHub para detalles exactos sobre opciones, versiones y actualizaciones.
