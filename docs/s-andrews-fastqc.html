FastQC

Repositorio en GitHub · 611 estrellas · Java

¿Qué es y para qué sirve?

FastQC es una herramienta de control de calidad diseñada para analizar datos de secuenciación de alta throughput (HTS), como los generados por plataformas como Illumina. Su objetivo principal es proporcionar un análisis rápido y automatizado que identifique posibles problemas en los datos de secuenciación, como baja calidad de las bases, contaminación, sobre-representación de secuencias o problemas de adaptadores. Es ampliamente utilizado como primer paso en cualquier flujo de análisis bioinformático para evaluar la calidad de los datos antes de proceder a alineamientos, ensamblajes o análisis posteriores.

Instalación

FastQC está escrito en Java, por lo que requiere un entorno Java 8 o superior instalado. Puedes instalarlo mediante el siguiente comando si tienes acceso a un gestor de paquetes como apt:

sudo apt update
sudo apt install fastqc

Alternativamente, puedes descargar el archivo ejecutable (JAR) directamente desde el repositorio oficial:

wget https://github.com/s-andrews/FastQC/releases/latest/download/FastQC.jar
chmod +x FastQC.jar

Uso básico

Una vez instalado, puedes ejecutar FastQC sobre uno o más archivos de secuenciación (por ejemplo, archivos FASTQ). El siguiente comando es ilustrativo; la sintaxis exacta puede variar según la versión y el sistema. Verifica siempre la documentación oficial para los comandos precisos:

java -jar FastQC.jar -o ./salida /ruta/a/tus/datos/*.fastq

Esto generará un archivo HTML con el informe de control de calidad para cada archivo de entrada.

Flujo de trabajo típico

El flujo de trabajo habitual con FastQC incluye los siguientes pasos:

  1. Preparar los archivos de secuenciación (FASTQ) en formato adecuado.
  2. Ejecutar FastQC sobre cada archivo o grupo de archivos.
  3. Revisar los informes HTML generados para identificar posibles problemas.
  4. Tomar decisiones de limpieza o filtrado basadas en los resultados (por ejemplo, usar herramientas como Trimmomatic si hay problemas con adaptadores).
  5. Proseguir con el siguiente paso del análisis (alineamiento, ensamblaje, etc.).

Errores comunes y buenas prácticas

  • Java no encontrado: Asegúrate de tener Java 8 o superior instalado. Puedes verificar con java -version.
  • Permisos de ejecución: Si usas el archivo JAR, no olvides dar permiso de ejecución con chmod +x.
  • Archivos de entrada incorrectos: FastQC espera archivos FASTQ. Si intentas analizar un archivo en formato BAM o SAM, fallará.
  • Resultados de calidad baja: Si el informe muestra muchas advertencias (como baja calidad en las bases o sobre-representación de secuencias), considera realizar limpieza previa con herramientas como Trimmomatic o Cutadapt.
  • Automatización: Puedes integrar FastQC en pipelines automatizados (por ejemplo, con Nextflow o Snakemake) para análisis en lote.
