variant-calling-pipeline-gatk4

Repositorio · Autores: 59 estrellas · Nextflow

¿Qué es y para qué sirve?

variant-calling-pipeline-gatk4 es un pipeline de llamado de variantes genómicas basado en GATK4 y Nextflow. Está diseñado para realizar el análisis de variantes en datos de secuenciación de alto rendimiento (NGS), siguiendo una metodología robusta y reproducible. El pipeline automatiza el flujo completo desde la alineación de lecturas hasta la identificación y filtrado de variantes, incluyendo la generación de archivos de resultados en formatos estándar como VCF.

Instalación

El pipeline se ejecuta mediante Nextflow, que requiere Java 8 o superior. Además, se necesitan los siguientes requisitos previos:

  • Nextflow: Instalación recomendada a través de curl -s https://get.nextflow.io | bash
  • Java: Instalar OpenJDK 8 o 11.
  • Dependencias de GATK4 (se manejan automáticamente mediante Nextflow).
curl -s https://get.nextflow.io | bash
chmod +x nextflow

Uso básico

Ejecutar el pipeline con un archivo de configuración y datos de entrada. Se recomienda usar nextflow run con un perfil específico (por ejemplo, local o singularity). 

nextflow run gencorefacility/variant-calling-pipeline-gatk4 \
  -profile local \
  --input "data/inputs.csv" \
  --genome "data/hg38.fasta" \
  --outDir "results/"

Flujo de trabajo típico

El flujo de trabajo sigue estas etapas:

  1. Validación y preprocesamiento de datos de entrada (lecturas FASTQ).
  2. Alineamiento de las lecturas a un genoma de referencia (usando herramientas como BWA o Bowtie2).
  3. Etiquetado de duplicados, corrección de errores de base, y ajuste de calidad.
  4. Llamado de variantes usando GATK4 (HaplotypeCaller).
  5. Filtrado de variantes y generación de resultados en formato VCF.

Errores comunes y buenas prácticas

  • Falta de memoria: GATK4 puede consumir mucho RAM. Asegúrate de asignar suficiente memoria al ejecutar el pipeline (usar --memory en Nextflow).
  • Genoma de referencia no compatible: Asegúrate de que el archivo de referencia (FASTA) esté indexado con bwa index y gatk CreateSequenceDictionary.
  • Configuración incorrecta del input: El archivo inputs.csv debe tener el formato esperado (ej: sample_id,fastq1,fastq2).
  • Usar perfiles: Si se ejecuta en un cluster, usa --profile slurm o --profile singularity para gestionar recursos adecuadamente.
  • Verificar la documentación oficial: La sintaxis exacta puede variar. Siempre revisa el repositorio para ver los parámetros específicos y ejemplos.
