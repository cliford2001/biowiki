Corrección de efectos de lote con Harmony

¿Qué es la corrección de efectos de lote y por qué importa?

En bioinformática, especialmente en análisis de secuenciación de ARN (RNA-seq), los efectos de lote (batch effects) son variaciones técnicas no biológicas que surgen debido a diferencias en los procesos experimentales: fechas de muestreo, reactivos, operadores, plataformas de secuenciación o condiciones de laboratorio. Estos efectos pueden distorsionar significativamente los resultados, haciendo que muestras biológicamente similares parezcan diferentes y viceversa.

La corrección de efectos de lote es una etapa crítica en el análisis de datos de expresión génica. Ignorarlos puede llevar a conclusiones erróneas, como la detección falsa de genes diferencialmente expresados o la pérdida de señales biológicas reales. Herramientas como Harmony han surgido como soluciones robustas para este desafío, permitiendo separar la variación biológica de la variación técnica.

¿Cómo funciona Harmony?

Harmony es un algoritmo basado en regularización de vecindad (neighborhood regularization) que se utiliza principalmente para ajustar datos de alta dimensionalidad, especialmente en análisis de células individuales (single-cell RNA-seq) y en RNA-seq de múltiples muestras. A diferencia de métodos como ComBat o limma, que operan en el espacio de expresión directo, Harmony trabaja en un espacio de vecindad (neighborhood space), donde cada celda o muestra se representa por su vecindad de puntos similares.

El proceso clave de Harmony consiste en:

  1. Construir una matriz de vecindad a partir de las distancias entre muestras (usando PCA o UMAP).
  2. Definir un factor de lote (batch factor) que indica qué grupo de muestras pertenece a cada lote.
  3. Ajustar los vectores de vecindad para minimizar la diferencia entre vecindades de muestras del mismo lote, mientras se preserva la estructura biológica.
  4. Proyectar de nuevo los datos al espacio original, ahora corregido del efecto de lote.

Este enfoque permite corregir el efecto de lote sin necesidad de modelar explícitamente las covariables técnicas, lo que lo hace más flexible y robusto en escenarios complejos.

Código de ejemplo: Corrección de efectos de lote con Harmony en R

Harmony se puede usar con datos de expresión obtenidos de herramientas como Seurat o Scanpy. A continuación, se muestra un ejemplo con R y el paquete harmony:

library(Seurat)
library(harmony)

# Supongamos que ya tenemos un objeto Seurat llamado "seurat_obj"
# con datos de expresión y un campo de lote en "orig.louvain" o similar

# 1. Normalización y PCA
seurat_obj <- NormalizeData(seurat_obj)
seurat_obj <- FindVariableFeatures(seurat_obj, nfeatures = 2000)
seurat_obj <- ScaleData(seurat_obj, features = VariableFeatures(seurat_obj))
seurat_obj <- RunPCA(seurat_obj, features = VariableFeatures(seurat_obj), verbose = FALSE)

# 2. Ajuste de efectos de lote con Harmony
seurat_obj <- RunHarmony(seurat_obj, group.by = "batch", reduction.key = "pca", dims = 1:30)

# 3. Visualización
seurat_obj <- RunUMAP(seurat_obj, reduction = "harmony", dims = 1:30)
DimPlot(seurat_obj, group.by = "batch")
DimPlot(seurat_obj, group.by = "cell_type")

Este código muestra cómo integrar Harmony en un flujo de trabajo típico de Seurat. El argumento group.by = "batch" especifica el campo que contiene la información de lote (puede ser "orig.louvain", "donor", "run", etc.).

Resumen

La corrección de efectos de lote es una necesidad fundamental en el análisis de datos biológicos, especialmente en el contexto de estudios multi-lote o multi-plataforma. Harmony ofrece una solución poderosa y eficaz al trabajar directamente en el espacio de vecindad, permitiendo preservar la variación biológica real mientras se corrige la variación técnica. Su flexibilidad y robustez lo convierten en una herramienta preferida en análisis de células individuales y en RNA-seq integrado. Su uso adecuado requiere una comprensión clara de los factores de lote y una validación post-corrección para asegurar que no se hayan eliminado señales biológicas reales.

Errores comunes y buenas prácticas

  • No identificar el factor de lote correctamente: Asegúrate de que el campo usado para group.by contiene realmente la información de lote (no confundas con cell_type o orig.louvain).
  • Aplicar Harmony antes del PCA o sin normalización: Siempre normaliza los datos antes de PCA, ya que Harmony opera sobre la matriz de vecindad construida a partir de PCA.
  • Ignorar la validación: Después de corregir el efecto de lote, verifica que los grupos biológicos se mantienen separados (por ejemplo, usando UMAP/PCA con colores por tipo celular).
  • Usar demasiados componentes: Aunque Harmony puede manejar hasta 30 dimensiones, usar un número muy alto puede sobreajustar. Comienza con 10–20 y ajusta si es necesario.
  • Confundir con otros métodos: Harmony no reemplaza a métodos de diferencial expresión como DESeq2 o edgeR. Corrige el lote antes del análisis diferencial.

Siempre documenta el proceso de corrección de lote para reproducibilidad.
