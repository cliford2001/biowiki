El formato VCF explicado

¿Qué es el formato VCF?

El formato VCF (Variant Call Format) es un estándar de texto plano ampliamente utilizado en bioinformática para almacenar información sobre variantes genéticas, especialmente variantes de un solo nucleótido (SNVs), inserciones, deleciones y otros tipos de variantes estructurales. Fue desarrollado por el 1000 Genomes Project y ahora es el formato de referencia para compartir resultados de llamado de variantes en estudios de genómica y proteómica.

La principal ventaja de VCF es su capacidad para representar de forma estructurada y legible una gran cantidad de información sobre variantes: desde la posición genómica y las bases alternativas, hasta cualidades de llamado, filtros, y anotaciones funcionales. Esto lo convierte en un pilar esencial en el análisis de datos genómicos, desde estudios de enfermedades hasta evolución y farmacogenómica.

Estructura del archivo VCF

Un archivo VCF está compuesto por dos secciones principales: el encabezado (##) y los datos de las variantes (líneas que comienzan con un cromosoma, posición y bases).

Encabezado

El encabezado contiene metadatos clave que definen la estructura del archivo. Cada línea comienza con ## y puede incluir:

  • ##fileformat=VCFv4.3: especifica la versión del formato.
  • ##contig: descripción de los cromosomas o contigs en el genoma de referencia.
  • ##INFO: define campos personalizados que almacenan información adicional sobre cada variante (por ejemplo, AC para número de copias de alelo alternativo).
  • ##FORMAT: describe cómo se estructuran los datos por muestra (por ejemplo, GT para genotipo, DP para profundidad).
  • ##FILTER: lista los filtros aplicados durante el llamado (como LowQual).

Cuerpo del archivo

Las líneas de datos comienzan con un cromosoma, seguido de la posición (1-based), ID de la variante (o . si no hay), la base principal (ref), la base alterna (alt), el cualidad (QUAL), el filtro (FILTER), y luego los campos FORMAT y datos por muestra.

chr1    12345   rs6789  G       A       30.5    PASS    GT:DP:GQ      0/1:12:25

En este ejemplo:

  • chr1: cromosoma
  • 12345: posición (1-based)
  • rs6789: identificador dbSNP
  • G: base de referencia
  • A: base alterna
  • 30.5: puntuación de calidad del llamado
  • PASS: el llamado pasó todos los filtros
  • GT:DP:GQ: campos de formato (genotipo, profundidad, calidad genotípica)
  • 0/1:12:25: datos para una muestra (heterocigoto, profundidad 12, calidad 25)

Herramientas comunes que usan VCF

El formato VCF es ampliamente soportado por herramientas de análisis genómico. Algunas de las más importantes incluyen:

  • BCFtools: herramienta de línea de comandos para manipular archivos VCF y BCF (formato binario de VCF).
  • vt: herramienta especializada para operaciones sobre VCF (filtros, combinación, normalización).
  • PLINK: para análisis de asociación (GWAS) con datos de variantes.
  • Annovar: para anotar variantes con información funcional (efectos sobre genes, variantes de interés, frecuencias en poblaciones).
  • GATK: en su pipeline de llamado de variantes (HaplotypeCaller), genera archivos VCF como salida.

Además, herramientas como bcftools annotate, bcftools filter, y bcftools view permiten procesar, filtrar y modificar archivos VCF de forma eficiente.

Errores comunes y buenas prácticas

El uso incorrecto del formato VCF puede llevar a errores graves en análisis posteriores. Aquí algunas prácticas clave:

Errores comunes

  • Usar cromosomas no estándar: asegurarse de que los cromosomas en el archivo coincidan con el genoma de referencia (por ejemplo, chr1 vs 1).
  • Ignorar la normalización: variantes como AC y AT en una región de repetición deben normalizarse para evitar duplicación de variantes.
  • Dejar campos vacíos sin sentido: evitar valores como 0/0 en datos no llamados o usar . para valores faltantes.
  • Combinar VCFs sin validar: los archivos de diferentes pipelines o genomas de referencia no deben combinarse sin normalización y validación previa.

Buenas prácticas

  • Usar bcftools norm -m - para normalizar variantes (elimina redundancias en regiones de repeticiones).
  • Validar el archivo con bcftools stats o vcf-validator antes de compartirlo.
  • Documentar los campos personalizados con ##INFO y ##FORMAT para mayor reproducibilidad.
  • Usar el genoma de referencia correcto y mantener consistencia entre el VCF y el índice de referencia.
  • Evitar almacenar datos sensibles (como identificadores de pacientes) en el archivo VCF.
