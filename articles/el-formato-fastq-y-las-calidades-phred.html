El formato FASTQ y las calidades Phred

Introducción: La base del análisis de secuenciación

El formato FASTQ es el estándar universal para almacenar datos de secuenciación de ADN y ARN de próxima generación (NGS). Su diseño simple pero robusto permite representar de forma eficiente tanto las secuencias de nucleótidos como su calidad estimada, un aspecto crítico en el análisis bioinformático moderno. Sin un manejo correcto de los datos FASTQ, cualquier análisis posterior —ya sea alineamiento, ensamblaje, o detección de variantes— puede verse comprometido por errores sistemáticos o pérdida de información.

La métrica clave que distingue al FASTQ de otros formatos de secuencias es la representación de las calidades Phred. Este sistema numérico permite cuantificar la confianza en cada base llamada, lo cual es esencial para aplicaciones como el filtrado de lecturas bajas calidad, el ajuste de parámetros de alineamiento, y la estimación de profundidad de cobertura. En este artículo, exploramos en profundidad la estructura del formato FASTQ, el significado y cálculo de las calidades Phred, y cómo herramientas como STAR, Salmon, kallisto, y samtools dependen de estos datos.

Estructura del formato FASTQ

Cada archivo FASTQ está compuesto por bloques de cuatro líneas consecutivas que representan una única lectura:

@READ_001
ATCGATCGATCGATCGATCGATCGATCGATCGATCGATCGATCGATCG
+
IIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIII

Donde:

  • @READ_001: Encabezado que identifica la lectura, generalmente incluye información del flujo de secuenciación y el número de la lectura.
  • ATCG...: Secuencia de nucleótidos. Puede estar en mayúsculas o minúsculas, pero las letras se interpretan como bases (A, T, C, G).
  • +: Línea de separación que marca el inicio de los valores de calidad.
  • IIII...: Cadenas de caracteres ASCII que representan los valores de calidad Phred.

Las calidades Phred: significado y cálculo

Las calidades Phred se calculan mediante la fórmula:

Q = -10 × log₁₀(P)

Donde P es la probabilidad de que la base llamada sea errónea. Así, un valor Q = 30 indica que la probabilidad de error es 1 en 1000 (10⁻³).

Los valores de calidad se almacenan como caracteres ASCII, no como números. Los caracteres se mapean a valores Q mediante el offset:

Q = ASCII_value - offset

El offset más común es 33 (Phred+33), aunque algunos formatos antiguos usan 64 (Phred+64). Las herramientas como fastp, sickle, y Trimmomatic requieren que el usuario especifique correctamente el offset para evitar errores en el filtrado.

Uso práctico en herramientas bioinformáticas

La calidad de las lecturas influye directamente en el rendimiento de las herramientas más utilizadas en el flujo de trabajo:

Alinhamiento con STAR

STAR permite especificar un umbral de calidad para descartar lecturas con calidades promedio muy bajas:

STAR --readFilesIn reads.fastq --outFileNamePrefix star_output/ --alignEndsProtruding EndToEnd --outFilterMismatchNmax 2

Además, se recomienda realizar un filtrado previo con fastp para eliminar lecturas de baja calidad antes del alineamiento.

Estimación de expresión con Salmon y kallisto

Estas herramientas de quantificación de expresión transcriptómica (transcriptome quantification) aprovechan las calidades para ajustar la probabilidad de asignación de lecturas a genes. kallisto, en particular, permite usar el archivo FASTQ con calidades Phred+33 directamente en su flujo de trabajo:

kallisto index -i transcriptome.idx transcriptome.fasta
kallisto quant -i transcriptome.idx -o output/ reads_1.fastq reads_2.fastq

El algoritmo internamente considera las calidades para mejorar la precisión de los estimados.

Filtrado con fastp

La herramienta fastp es ampliamente usada para filtrar y adaptar archivos FASTQ:

fastp -i reads.fastq -o filtered.fastq -q 20 -u 50 -l 30 -h report.html

Este comando filtra lecturas con calidades promedio < 20, elimina secuencias con más de 50 bases de baja calidad, y mantiene solo lecturas >30 pares.

Resumen

El formato FASTQ es el cimiento del análisis de secuenciación NGS. Su capacidad para almacenar tanto secuencias como calidades Phred de forma compacta y estandarizada lo convierte en una pieza fundamental en todo pipeline bioinformático. Las calidades Phred, calculadas a partir de la probabilidad de error, permiten realizar decisiones informadas sobre el tratamiento de datos, desde el filtrado hasta la asignación de lecturas. Herramientas como STAR, Salmon, kallisto, fastp, y samtools dependen críticamente de que los datos FASTQ estén correctamente formateados y con un offset de calidad adecuado. Ignorar el manejo de las calidades Phred puede llevar a resultados erróneos, sesgos en la expresión génica, o pérdida de datos valiosos.

Errores comunes y buenas prácticas

  • No verificar el offset de calidad: Usar un offset incorrecto (33 vs 64) provoca que las calidades se interpreten erróneamente. Siempre verifica el origen de tus datos y usa herramientas como fastq-dump (SRA Toolkit) o zcat para inspeccionar el primer caracter.
  • Ignorar el filtrado previo: Enviar datos con muchas lecturas de baja calidad a herramientas de alineamiento o cuantificación puede ralentizar el proceso y degradar los resultados.
  • Usar un solo valor de calidad como corte absoluto: En lugar de fijar un umbral fijo, considera el promedio de calidad por lectura y la distribución de calidades, lo cual puede detectar problemas más sutiles (por ejemplo, en el extremo 3’).
  • Guardar datos sin calidad: Siempre conserva los archivos FASTQ originales, y si realizas transformaciones, documenta los cambios en un archivo de metadatos.
  • Usar herramientas que no soportan Phred+33: Aunque raro, algunas herramientas antiguas o no mantenidas esperan Phred+64. Verifica la documentación de cada herramienta.
