Validación de variantes de nuevo descubrimiento con VEP y dbNSFP

Introducción

En el análisis de variantes genéticas, especialmente en estudios de descubrimiento de nuevas variantes (novel variants), la anotación funcional es un paso crítico para evaluar su posible impacto biológico. Las herramientas VEP (Variant Effect Predictor) y dbNSFP son fundamentales en este proceso. VEP proporciona una anotación detallada de los efectos de las variantes sobre genes y enzimas, mientras que dbNSFP integra información de múltiples fuentes sobre el impacto funcional, conservación, y asociación con enfermedades.

La validación de variantes de nuevo descubrimiento no se limita a su identificación, sino que requiere una evaluación rigurosa de su potencial patogenicidad. En este artículo, explicaremos cómo combinar VEP y dbNSFP para validar variantes recién descubiertas, asegurando precisión, reproducibilidad y coherencia con la evidencia científica actual.

Preparación de los datos

Antes de anotar las variantes, es esencial tener un archivo de variantes en formato BCF o VCF (variant call format), que contenga al menos las columnas necesarias: CHROM, POS, ID, REF, ALT, y INFO. Si se trabaja con variantes de nuevo descubrimiento, es recomendable filtrar por calidad (por ejemplo, QUAL > 30), frecuencia en poblaciones (como gnomAD), y soporte de lectura (DP).

bcftools view -i 'QUAL > 30 && AF < 0.01' raw_variants.vcf.gz -O z -o filtered_variants.vcf.gz

Análisis con VEP

VEP es una herramienta de la EBI (European Bioinformatics Institute) que anota variantes con información funcional, incluyendo efectos sobre genes (síntesis de proteína, splicing), consecuencias (missense, nonsense, frameshift), y puntuaciones de impacto como CADD.

Para anotar variantes, se requiere un archivo de entrada VCF (o BCF) y una base de datos local o en línea. Se recomienda usar una base de datos local para mayor eficiencia.

vep --input_file filtered_variants.vcf.gz \
    --output_file annotated_variants.vcf \
    --cache --dir /path/to/vep/cache \
    --format vcf --sift b --polyphen b --symbol --hgvs --af --af_1000g --af_gnomad \
    --minimal --force_overwrite

Las opciones clave incluyen:

  • --cache: Usa la base de datos de VEP local.
  • --format vcf: Mantiene el formato VCF.
  • --sift b y --polyphen b: Anota puntuaciones SIFT y PolyPhen.
  • --af_1000g y --af_gnomad: Incluye frecuencias en poblaciones.
  • --minimal: Reduce la cantidad de columnas.

Integración con dbNSFP

dbNSFP es una base de datos que integra múltiples fuentes de información funcional, como CADD, REVEL, FATHMM, LRT, y más. Es especialmente útil para evaluar variantes con bajo impacto fenotípico, que pueden no ser detectadas por VEP solo.

Para integrar dbNSFP, primero se debe obtener el archivo de dbNSFP en formato VCF (por ejemplo, dbNSFP.vcf.gz), y luego usar bcftools para unirlo con el archivo anotado por VEP.

bcftools annotate -a dbNSFP.vcf.gz -c "CHROM,POS,REF,ALT,REVEL,PROTEIN_CHANGE,CADD,GERP++RS,phyloP,1000Gp3_AF,gnomAD_AF" -h -o annotated_with_dbnsfp.vcf.gz filtered_variants.vcf.gz

Este comando añade las columnas seleccionadas de dbNSFP al archivo VCF. Es importante verificar que los campos coincidan (por ejemplo, CHROM, POS, REF, ALT).

Evaluación y validación

Una vez anotado, el análisis final consiste en filtrar variantes según criterios de interés. Por ejemplo, se pueden seleccionar variantes con:

  • Impacto funcional alto: Consequence = "stop_gained", frameshift_variant, nonsense_mediated_decay.
  • Puntuación REVEL > 0.5: indicador de patogenicidad.
  • CADD > 20: alto impacto predictivo.
  • Frecuencia en poblaciones baja: gnomAD_AF < 0.001.

En R, se puede usar vcfR para cargar y filtrar los datos:

library(vcfR)

# Cargar el archivo
vcf <- read.vcfR("annotated_with_dbnsfp.vcf.gz")

# Filtrar
filtered <- vcfR::select.vcfR(vcf, 
  filter = "INFO_REVEL > 0.5 && INFO_CADD > 20 && INFO_gnomAD_AF < 0.001")

# Guardar
write.vcfR(filtered, "high_impact_variants.vcf.gz")

Resumen

La validación de variantes de nuevo descubrimiento requiere una combinación de herramientas robustas y datos actualizados. VEP proporciona una anotación funcional detallada, mientras que dbNSFP aporta puntuaciones predictivas basadas en múltiples modelos. Juntas, estas herramientas permiten identificar variantes de alto impacto, con baja frecuencia en poblaciones y alta puntuación de patogenicidad, facilitando la selección de candidatas para estudios funcionales posteriores.

Errores comunes / buenas prácticas

  • No usar bases de datos actualizadas: siempre verificar que VEP y dbNSFP estén actualizados (se recomienda usar la última versión).
  • Ignorar frecuencias poblacionales: variantes con alta frecuencia en gnomAD o 1000 Genomes rara vez son patógenas de novo.
  • Confundir CADD con REVEL: CADD es una puntuación de daño, REVEL es una puntuación de patogenicidad; usar ambas con cautela.
  • Usar datos sin filtrar: siempre aplicar filtros de calidad antes de anotar.
  • Combinar bases de datos sin validar coincidencia de coordenadas: usar bcftools merge o vcf-merge para asegurar consistencia.
