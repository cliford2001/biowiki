Análisis de expresión diferencial con DESeq2

Introducción

El análisis de expresión diferencial (DE) es una de las tareas fundamentales en bioinformática funcional, especialmente en estudios transcriptómicos. Su objetivo consiste en identificar genes cuya expresión se modifica significativamente entre diferentes condiciones biológicas, como tratamientos, enfermedades o estados celulares. DESeq2, desarrollado por Michael I. Love, Wolfgang Huber y Simon Anders, es uno de los paquetes más ampliamente utilizados para este fin en R, especialmente para datos de secuenciación de ARN (RNA-seq).

DESeq2 combina modelos estadísticos robustos con métodos de ajuste de dispersión, lo que permite manejar eficazmente la variabilidad biológica y técnica en los datos. Su enfoque basado en la distribución de negative binomial y el uso de estimadores empíricos de dispersión lo distingue de herramientas anteriores como edgeR y limma-voom. Además, ofrece una API clara y extensa, con funciones para visualización, filtrado de genes y validación de resultados.

Desarrollo

Preprocesamiento y entrada de datos

Antes de aplicar DESeq2, es esencial contar con un conteo de reads por gen, obtenido a partir de archivos de alineación (por ejemplo, STAR o kallisto). El paquete tximport facilita la importación de resultados de herramientas como Salmon o StringTie en un formato compatible con DESeq2.

library(tximport)
library(DESeq2)

# Importar datos de Salmon
files <- list.files("./salmon_output", full.names = TRUE)
tx2gene <- read.table("./data/tx2gene.txt", header = FALSE, stringsAsFactors = FALSE)

txi <- tximport(files, type = "salmon", tx2gene = tx2gene)

Construcción del diseño experimental y análisis

El primer paso en DESeq2 es crear un objeto de diseño experimental. Esto implica definir el diseño de factores (como tratamiento, tiempo, genotipo) y el grupo de comparación.

colData <- data.frame(
    condition = factor(c("control", "treated", "control", "treated")),
    batch = factor(c(1, 1, 2, 2))
)

dds <- DESeqDataSetFromTximport(txi, colData, design = ~ batch + condition)

El modelo incluye un efecto de batch para controlar variaciones técnicas entre diferentes secuencias de laboratorio.

Estimación de dispersión y prueba estadística

DESeq2 ajusta la dispersión de los datos usando un modelo que combina información de todos los genes. La función DESeq() realiza esta estimación y realiza la prueba de hipótesis para cada gen.

dds <- DESeq(dds)

Este paso genera un objeto con todos los parámetros estimados, incluyendo la dispersión, los efectos de los factores y los valores p.

Extracción de resultados y filtrado

Se pueden extraer los resultados con results(), especificando el factor y el nivel de comparación de interés.

res <- results(dds, name = "condition_treated_vs_control")

# Filtrar genes con valor p ajustado < 0.05 y log2 fold change > 1
res_sig <- subset(res, padj < 0.05 & abs(log2FoldChange) > 1)

Visualización de resultados

DESeq2 incluye funciones de visualización integradas. Por ejemplo, el MA plot muestra la relación entre el fold change y la expresión media, destacando genes diferencialmente expresados.

plotMA(res, alpha = 0.05)

Los volcano plots también son útiles para visualizar la relación entre el log2 fold change y el valor p ajustado.

Resumen

El análisis de expresión diferencial con DESeq2 es una metodología robusta, estadísticamente rigurosa y ampliamente validada. Su capacidad para modelar la variabilidad biológica y técnica, junto con su integración con flujos de trabajo modernos de secuenciación de ARN, lo convierte en una herramienta esencial en cualquier proyecto de transcriptómica. A través de un diseño experimental bien definido, estimación de dispersión empírica y pruebas estadísticas ajustadas, DESeq2 permite identificar genes diferencialmente expresados con alta confianza y reproducibilidad.

Errores comunes / buenas prácticas

  • No usar datos raw de conteo sin validación: Asegúrate de que los archivos de conteo sean correctos y representen datos de secuenciación de ARN (no de otros tipos).
  • Ignorar el diseño experimental: El diseño en design = ~ factor debe reflejar la estructura del experimento, incluyendo factores de confusión como batch.
  • Usar valores p sin ajuste: Siempre trabaja con valores p ajustados (padj) para controlar el error de tipo I.
  • Filtrar genes antes del análisis: Es recomendable filtrar genes con muy baja expresión (por ejemplo, count < 10 en al menos 3 muestras) antes de aplicar DESeq2 para mejorar la sensibilidad.
  • Visualizar resultados: Usa plotMA(), plotDispEsts() y vsn() para validar el modelo y detectar anomalías.
