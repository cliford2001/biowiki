Modelado de interacciones entre genes y reguladores con chromVAR y cisTarget

Introducción: ¿Por qué entender las interacciones reguladoras?

En biología del desarrollo y en enfermedades como el cáncer, el control de la expresión génica no reside únicamente en los genes codificadores, sino en un complejo sistema de regulación que involucra factores de transcripción (TFs), elementos promotores y elementos reguladores distantes. Comprender cómo los factores de transcripción interactúan con el ADN para modular la expresión génica es clave para desentrañar mecanismos celulares subyacentes. Las tecnologías de secuenciación de chromatinas abiertas, como ATAC-seq, proporcionan una ventana para explorar esta regulación. Sin embargo, el paso crítico de inferir qué TFs están activos y cuáles genes regulan requiere herramientas especializadas. En este artículo, exploramos el uso combinado de chromVAR y cisTarget para modelar interacciones entre genes y reguladores a partir de datos de ATAC-seq.

Primer paso: Análisis de variabilidad en accesibilidad cromatínica con chromVAR

chromVAR es un paquete de R diseñado para detectar variabilidad en la accesibilidad del cromatina entre muestras, específicamente para identificar si un conjunto de regiones reguladoras (como promotores o enhancers) es más accesible en ciertos estados celulares. El enfoque principal de chromVAR se basa en el cálculo de la variabilidad de la accesibilidad cromatínica (chromVAR score), que mide la diferencia en la accesibilidad promedio entre regiones asociadas a un determinado factor de transcripción (TF) y el resto del genoma.

Proceso clave:

  • Se define un motivo de TF (TFBS) como una secuencia de ADN que se espera que sea reconocida por un factor de transcripción específico.
  • Se construye una matriz de accesibilidad cromatínica desde datos de ATAC-seq (por ejemplo, usando Signac o ArchR).
  • Para cada TF, se asignan regiones del genoma que contienen su motivo a un conjunto de regiones asociadas a ese TF.
  • chromVAR calcula el chromVAR score como el promedio de accesibilidad de estas regiones, ajustado por el promedio general del genoma.
  • Una puntuación alta indica que las regiones asociadas a ese TF son más accesibles en esa muestra, lo que sugiere que el TF podría estar activo.
library(chromVAR)
library(Seurat)

# Supongamos que 'atac_matrix' es la matriz de accesibilidad (genes x muestras)
# y 'tf_motifs' es un objeto de tipo TFBSList

# Calcular chromVAR scores
chromvar_scores <- chromVAR(
  countMatrix = atac_matrix,
  motifMatrix = tf_motifs,
  min.cells = 10,
  min.regions = 10
)

# Obtener scores por muestra y TF
scores <- getChromVARScore(chromvar_scores)

Segundo paso: Inferencia de reguladores con cisTarget

Mientras que chromVAR identifica qué TFs podrían estar activos, cisTarget va un paso más allá: infiere qué genes son regulados por esos TFs. cisTarget utiliza una base de datos de motivos de TFs y sus interacciones conocidas con genes, combinada con datos de ATAC-seq, para predecir redes regulatorias.

Flujo de análisis:

  1. Se ingresa el conjunto de regiones de ATAC-seq y el motivo de TF asociado.
  2. Se identifican las regiones activas en la muestra.
  3. Se buscan genes cercanos (por ejemplo, dentro de 100 kb) a las regiones activas que contienen motivos de TF.
  4. Se calcula una puntuación de regulación (c-score) para cada TF-gene, que refleja la probabilidad de que el TF regule ese gen.
  5. Se filtran resultados por significancia estadística (p-valor ajustado).
library(cisTarget)

# Supongamos que 'peaks' es un objeto de tipo GRanges con picos de ATAC-seq
# y 'tf_motifs' es una lista de motivos

# Crear una base de datos de regulación
regulatory_network <- cisTarget(
  peaks = peaks,
  tf_motifs = tf_motifs,
  gene_annotation = gene_annotations,
  upstream = 100000,
  downstream = 100000,
  use_differential = TRUE
)

# Obtener los genes regulados por TFs con alta puntuación
regulated_genes <- getRegulatedGenes(regulatory_network, threshold = 0.01)

La salida de cisTarget es una red de regulación que puede visualizarse con herramientas como igraph o integrarse con Seurat para análisis de grupos celulares.

Integración de chromVAR y cisTarget: un flujo completo

La verdadera potencia surge al combinar ambos enfoques. En una pipeline típica:

  1. Se procesa el ATAC-seq con herramientas como macs2 o csaw para detectar picos.
  2. Se calculan los chromVAR scores para identificar TFs activos en cada estado celular.
  3. Con los TFs activos, se usa cisTarget para inferir cuáles genes están regulados por ellos.
  4. Finalmente, se valida la red reguladora con datos de expresión génica (por ejemplo, desde scRNA-seq) o con experimentos funcionales.

Este enfoque permite no solo identificar TFs activos, sino también construir modelos de regulación que conectan factores de transcripción con genes diana, facilitando la generación de hipótesis funcionales para experimentos posteriores.

Resumen

El modelado de interacciones entre genes y reguladores es una pieza fundamental del análisis de datos de cromatina. chromVAR permite identificar factores de transcripción con actividad potencial en muestras específicas mediante el análisis de variabilidad en la accesibilidad cromatínica. cisTarget, por su parte, infiere las relaciones regulatorias entre TFs y genes diana a partir de datos de ATAC-seq. Cuando se combinan, estas herramientas permiten reconstruir redes regulatorias dinámicas, ofreciendo una visión detallada de cómo se controla la expresión génica en diferentes contextos biológicos.

Errores comunes y buenas prácticas

  • Usar motivos sin validación: Asegúrese de que los motivos de TFs provengan de fuentes confiables como JASPAR, HOCOMOCO o cis-BP, y no se basen en motivos sintéticos.
  • Ignorar el contexto espacial: Los genes regulados por un TF pueden no estar cerca de las regiones de ATAC-seq. Siempre considere el contexto genómico y use criterios razonables de distancia (por ejemplo, 100 kb).
  • No validar con expresión génica: Las predicciones de cisTarget son hipotéticas. Valide los genes diana con datos de expresión de scRNA-seq o con análisis de correlación.
  • Overfitting: Evite usar demasiados TFs o regiones. Use filtros estadísticos y ajuste el umbral de puntuación según el tamaño del dataset.
  • No considerar la heterogeneidad celular: Si el ATAC-seq es de una población mixta, los resultados pueden estar sesgados. Prefiera datos de células aisladas o use métodos como ArchR o Signac para análisis de células individuales.
