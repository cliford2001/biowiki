GATK best practices para variant calling

Introducción

El Genome Analysis Toolkit (GATK) es una de las plataformas más utilizadas en bioinformática para el análisis de variantes genómicas, especialmente para variantes de un solo nucleótido (SNVs) y variantes de inserción/deleción (indels). Su uso se ha extendido ampliamente en estudios de genómica de cáncer, enfermedades hereditarias, y poblaciones. Sin embargo, el éxito de un llamado de variantes (variant calling) depende críticamente de seguir las best practices establecidas por el equipo de GATK.

Este artículo explora las prácticas recomendadas para realizar llamados de variantes con GATK, desde la preparación de datos hasta el filtrado final, con el objetivo de maximizar la precisión y la reproducibilidad. Ignorar estos pasos puede generar falsos positivos, falsos negativos o sesgos sistemáticos en los resultados.

Preparación de los datos de entrada

Antes de ejecutar GATK, es fundamental preparar los archivos de alineamiento (BAM) adecuadamente. El proceso típico incluye:

  • Reparación de alineamientos dañados o incompletos con samtools fixmate o picard FixMateInformation.
  • Marcar duplicados de PCR con picard MarkDuplicates para evitar sobreestimación de variantes.
  • Reajuste de los mapas de alineamiento para corregir desviaciones en el alineamiento (realigning) con gatk RealignerTargetCreator y gatk IndelRealigner (si se usa GATK versión anterior a 4.x).
  • Para GATK 4.x, se recomienda usar gatk BaseRecalibrator y gatk ApplyBQSR para corregir la calidad de base sesgada por artefactos de secuenciación.

Variant Calling con HaplotypeCaller

El algoritmo HaplotypeCaller es el estándar de facto para variant calling en GATK 4.x. A diferencia de métodos anteriores que llamaban variantes localmente, HaplotypeCaller usa una estrategia de reconstrucción de haplotipos que mejora la detección de indels y SNVs en regiones complejas.

Un comando típico de llamado de variantes es: 

gatk HaplotypeCaller \
  -I input.bam \
  -R reference.fasta \
  -O output.vcf \
  --emit-ref-confidence GVCF \
  -stand-call-conf 30 \
  -pcr-indel-tolerance 10

Algunos parámetros clave:

  • --emit-ref-confidence GVCF: Emite llamados de referencia en formato GVCF para posterior genotipado conjunto (joint genotyping).
  • -stand-call-conf 30: Umbral de confianza para llamar variantes (30 es recomendado para alto fidelidad).
  • -pcr-indel-tolerance: Aumenta tolerancia a errores de PCR en indels (útil en regiones con repetición).

Post-procesamiento y filtrado

Tras el llamado, es esencial filtrar los resultados. El primer paso es usar gatk VariantFiltration para aplicar reglas basadas en calidad del llamado, profundidad, y distribución de alelos. 

gatk VariantFiltration \
  -V input.vcf \
  -O filtered.vcf \
  --filter-expression "QD < 2.0" --filter-name "LowQD" \
  --filter-expression "FS > 60.0" --filter-name "StrandBias" \
  --filter-expression "MQ < 40.0" --filter-name "LowMQ" \
  --filter-expression "SOR > 3.0" --filter-name "StrandOddsRatio"

Además, se recomienda realizar un joint genotyping cuando se analizan múltiples muestras, usando gatk GenotypeGVCFs sobre todos los GVCFs generados.

Resumen

Para un llamado de variantes robusto y reproducible con GATK, es esencial:

  • Preparar los datos de entrada con corrección de alineamientos, marcado de duplicados y recalibración de base.
  • Usar HaplotypeCaller en modo GVCF para permitir genotipado conjunto.
  • Filtrar variantes con criterios bien definidos basados en métricas como QD, FS, MQ y SOR.
  • Evitar configuraciones por defecto sin ajustar umbrals de confianza.

Errores comunes y buenas prácticas

Errores comunes

  • No aplicar BQSR: Ignorar la recalibración de base genera llamados de variantes con alto sesgo en calidad.
  • Usar llamado de variantes sin filtrado: Puede llevar a resultados contaminados por artefactos.
  • Configurar valores por defecto sin ajustar: Por ejemplo, usar stand-call-conf 10 en lugar de 30 puede aumentar falsos positivos.
  • Ignorar el uso de GVCF: En estudios de múltiples muestras, omitir el GVCF reduce la capacidad de detección de variantes raras.

Buenas prácticas

  • Utilizar gatk ValidateSamFile para verificar la integridad de los archivos BAM.
  • Verificar que el índice de referencia (FASTA) y el archivo de variantes de referencia (como dbsnp) estén actualizados.
  • Documentar cada paso con comandos y versiones de herramientas usadas.
  • Validar resultados con herramientas como bcftools o vcftools para verificar calidad de VCF.
