Detección de variantes de baja frecuencia con DeepVariant y variant calling pipelines

Introducción

La detección de variantes de baja frecuencia (VAF, por sus siglas en inglés) es un reto clave en bioinformática, especialmente en estudios de cáncer, enfermedades genéticas raras y microbiología. Estas variantes, que pueden estar presentes en menos del 5% de las lecturas alineadas, son críticas para entender la heterogeneidad tumoral, la resistencia a fármacos o la composición de microbiomas. En este artículo, exploramos cómo herramientas como DeepVariant y pipelines convencionales (como GATK) permiten la identificación precisa de estos cambios, abordando sus fortalezas, limitaciones y mejores prácticas.

Principios de la detección de variantes de baja frecuencia

Las variantes de baja frecuencia se definen como alteraciones genéticas presentes en una fracción baja del material genético de una muestra. Su detección requiere un enfoque altamente sensible, ya que los errores de secuenciación o de alineamiento pueden enmascarar señales reales. La clave está en el balance entre sensibilidad (detectar variantes reales) y especificidad (evitar falsos positivos).

Desafíos técnicos

  • Errores de secuenciación: los errores aleatorios en las lecturas pueden simular variantes reales.
  • Complejidad de la región: zonas repetitivas o con homología alta dificultan el alineamiento preciso.
  • Profundidad de cobertura: muestras con baja profundidad de secuenciación (por ejemplo, < 20x) aumentan el riesgo de pérdida de variantes.
  • Pruebas múltiples: el uso de múltiples algoritmos puede generar inconsistencias si no se validan.

DeepVariant: aprendizaje automático para variantes

DeepVariant, desarrollado por Google, utiliza redes neuronales convolucionales (CNN) para identificar variantes. En lugar de depender exclusivamente de reglas heurísticas, DeepVariant convierte cada sitio de secuenciación en una imagen de características ("image"), donde cada pixel representa un alineamiento en un contexto de 100 nucleótidos. Esta abordaje permite modelar patrones complejos de cobertura, errores y variación.

Fortalezas para variantes de baja frecuencia

  • Alta especificidad: reduce falsos positivos gracias al aprendizaje profundo.
  • Robustez ante errores: capaz de distinguir entre variaciones reales y artefactos.
  • Escalabilidad: puede procesar múltiples muestras con mínima configuración.

Uso con datos de baja frecuencia

Para detectar variantes de baja frecuencia, es recomendable usar DeepVariant con datos de alta profundidad (>100x). El modelo se entrena en datos de alta calidad, por lo que su desempeño mejora significativamente con más cobertura. Además, es crucial preprocesar los datos con herramientas como gatk PreprocessIntervals o picard MarkDuplicates para minimizar artefactos.

docker run -v $PWD:/data -w /data \
  google/deepvariant:latest \
  /opt/deepvariant/bin/run_deepvariant \
  --model_type=SNP_INDEL \
  --ref=GRCh38.fa \
  --reads=aligned_sorted.bam \
  --output_vcf=variants.vcf.gz \
  --num_shards=8

Pipelines convencionales: GATK y variant calling clásico

Los pipelines tradicionales, como los basados en GATK (Genome Analysis Toolkit), siguen un enfoque de dos pasos: recalibración de calidad (Base Quality Score Recalibration, BQSR) y llamado de variantes (HaplotypeCaller).

Proceso típico con GATK

  1. gatk SamToFastq para convertir BAM a FASTQ (si es necesario).
  2. gatk RealignerTargetCreator y gatk IndelRealigner para corregir alineamientos en regiones con inserciones/deleciones.
  3. gatk BaseQualityScoreRecalibrator para ajustar puntuaciones de calidad.
  4. gatk HaplotypeCaller para detectar variantes (en modo GVCF).
  5. gatk CombineGVCFs y gatk GenotypeGVCFs para obtener el VCF final.
gatk HaplotypeCaller \
  -R GRCh38.fa \
  -I aligned_sorted.bam \
  -O variants.g.vcf.gz \
  --emit-ref-confidence GVCF \
  -stand-call-conf 10

El parámetro --stand-call-conf 10 permite llamar variantes incluso con baja confianza, útil para variantes de baja frecuencia, aunque aumenta el riesgo de falsos positivos.

Validación y filtrado de variantes

Una vez obtenido el VCF, es esencial filtrar variantes potenciales. Las herramientas bcftools y vcftools permiten aplicar filtros basados en profundidad, frecuencia de alelos, calidad y otros criterios.

bcftools view -i "AF > 0.01" variants.vcf.gz | \
  bcftools filter -e "INFO/DP < 20" -Oz -o filtered.vcf.gz

Este comando filtra variantes con frecuencia alélica mayor a 1% y profundidad de cobertura mínima de 20x.

Resumen

La detección de variantes de baja frecuencia requiere un pipeline cuidadosamente diseñado. DeepVariant ofrece una solución altamente sensible basada en aprendizaje automático, ideal para datos de alta profundidad. Por otro lado, GATK sigue siendo una herramienta robusta y ampliamente validada, especialmente cuando se combinan con filtros rigurosos. La elección entre ambos depende del contexto: DeepVariant para alta sensibilidad y GATK para integración en pipelines existentes.

Errores comunes / buenas prácticas

  • NO usar DeepVariant con datos de baja profundidad: sus modelos están entrenados en datos de alta cobertura. Resultados pobres si se usan en secuencias < 30x.
  • Evitar el uso de parámetros de llamado sin filtrado: stand-call-conf 0 puede generar miles de falsos positivos.
  • Validar con herramientas externas: comparar resultados entre DeepVariant y GATK (o FreeBayes) mejora la confianza.
  • Usar vcf2fasta o VEP para anotación funcional: no asumir que una variante es relevante sin anotarla.
  • Documentar el pipeline completo: incluir versiones de herramientas, comandos y parámetros para reproducibilidad.
