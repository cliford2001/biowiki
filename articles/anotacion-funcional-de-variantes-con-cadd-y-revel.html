Anotación funcional de variantes con CADD y REVEL

Introducción: ¿Por qué anotar variantes funcionales?

En bioinformática, la identificación de variantes genéticas asociadas a enfermedades o trastornos es un paso fundamental en la genómica médica. Sin embargo, la mayoría de las variantes identificadas en secuenciación de exoma o genoma completo son de significado funcional incierto. La anotación funcional permite predecir el impacto potencial de una variante en la función del gen o proteína, lo que ayuda a priorizar candidatos para estudios clínicos o funcionales.

Dos herramientas altamente utilizadas en este contexto son CADD (Combined Annotation Dependent Depletion) y REVEL (Rare Exome Variant Ensemble Learner). Ambas aprovechan datos genómicos, conservación evolutiva, y modelos de aprendizaje automático para asignar una puntuación de daño a variantes, facilitando la evaluación de su relevancia biológica.

CADD: Puntuación de daño basada en datos genómicos

CADD es una herramienta que combina múltiples fuentes de anotación funcional en un único score que mide el daño potencial de una variante. El modelo se entrena usando variantes conocidas y no conocidas en el genoma humano, evaluando su posición, tipo de mutación, conservación evolutiva (como por ejemplo con PhyloP o SiPhy), y otras características funcionales.

La puntuación CADD está escalada de forma que valores más altos indican un mayor daño. Por ejemplo, una puntuación de 20 representa un nivel de daño que es más raro que el 99.9% de las variantes en el genoma humano. Aunque CADD no predice directamente el fenotipo, es una herramienta poderosa para filtrar variantes de interés.

# Descargar archivos de CADD (por ejemplo, para exoma humano)
wget https://krishna.stanford.edu/~cadd/v1.6/GRCh38/Exome/Exome_CADD_scores.tar.gz

# Ejemplo de uso en Python con un archivo VCF
import pandas as pd

# Cargar puntuaciones CADD
cadd_scores = pd.read_csv("cadd_scores.tsv", sep='\t')

# Filtrar variantes con CADD > 20
high_impact = cadd_scores[cadd_scores['CADD'] > 20]
print(f"Variantes con CADD > 20: {len(high_impact)}")

REVEL: Predicción de patogenicidad mediante ensamblaje

REVEL es un modelo de ensamblaje que combina puntuaciones de múltiples herramientas predictivas de patogenicidad, incluyendo SIFT, PolyPhen-2, CADD, y otras, para mejorar la precisión en la predicción de variantes patogénicas, especialmente en exomas de pacientes con enfermedades raras.

REVEL utiliza un enfoque de aprendizaje automático supervisado entrenado con variantes de alto y bajo impacto, y es particularmente útil para variantes raras que no están bien representadas en bases de datos como dbSNP o gnomAD. Su resultado es una puntuación entre 0 (no patogénica) y 1 (altamente patogénica).

Una puntuación de REVEL mayor a 0.7 se considera fuertemente patogénica en muchos estudios clínicos.

# Descargar modelo y puntuaciones REVEL
wget https://github.com/REVEL/REVEL/releases/download/v1.3/REVEL_1.3.zip
unzip REVEL_1.3.zip

# Ejemplo en R con el paquete 'REVEL'
library(REVEL)

# Cargar datos de variantes
variants <- read.table("variantes.vcf", header=TRUE, sep="\t")

# Calcular puntuación REVEL
revel_scores <- predictREVEL(variants, model="1.3")

# Filtrar variantes con REVEL > 0.7
high_revel <- variants[revel_scores > 0.7, ]

Integración de CADD y REVEL en el flujo de análisis

La combinación de CADD y REVEL permite un análisis más robusto. Por ejemplo, se puede priorizar variantes con:

  • CADD > 20 y REVEL > 0.7
  • CADD > 15 o REVEL > 0.6 (para filtros más flexibles)

Este enfoque permite equilibrar sensibilidad y especificidad, y es especialmente útil en estudios de enfermedades mendelianas, trastornos neurológicos, o cáncer hereditario.

# Combinar CADD y REVEL en Python
merged_data = pd.merge(cadd_scores, revel_scores, on=["CHROM", "POS", "REF", "ALT"])

# Filtro combinado
combined_filter = merged_data[(merged_data["CADD"] > 20) & (merged_data["REVEL"] > 0.7)]
print(f"Variantes con CADD > 20 y REVEL > 0.7: {len(combined_filter)}")

Resumen

La anotación funcional de variantes con CADD y REVEL es una práctica esencial en el análisis de datos genómicos. CADD proporciona un score de daño basado en datos genómicos y conservación, mientras que REVEL utiliza un modelo de ensamblaje para predecir patogenicidad con alta precisión. Juntas, estas herramientas permiten priorizar variantes con mayor impacto biológico, acelerando la identificación de causas genéticas de enfermedades.

Errores comunes / buenas prácticas

  • No confundir puntuaciones con resultados clínicos: CADD y REVEL son predictivos, no diagnósticos. Siempre validar con estudios funcionales o datos clínicos.
  • Usar el modelo correcto: CADD tiene versiones para exoma y genoma; asegurarse de usar la adecuada (por ejemplo, Exome_CADD_scores.tar.gz para exomas).
  • Evitar filtros excesivos: Un umbral demasiado alto (por ejemplo, CADD > 30) puede eliminar variantes reales de interés.
  • Combinar con otras fuentes: Integrar con datos de frecuencia en gnomAD, consenso de expertos (ClinVar), o herramientas como VEP (Variant Effect Predictor) para un análisis más completo.
  • Documentar el proceso: Registrar qué herramientas, versiones y umbrales se usaron, para reproducibilidad.
