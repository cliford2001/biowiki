Integración de datos de secuenciación de doble cadena con Kallisto y Salmon

Introducción: ¿Por qué integrar datos de secuenciación de doble cadena?

La secuenciación de ARN de doble cadena (dRNA-seq) ofrece una visión detallada de la transcripción, incluyendo tanto el ARN mensajero (mRNA) como el ARN no codificante (ncRNA), así como el inicio y final de transcripciones. Sin embargo, la integración de datos de dRNA-seq con otros tipos de datos transcriptómicos (como RNA-seq convencional, scRNA-seq o datos de proteómica) requiere una estrategia de cuantificación precisa y coherente. Esta es donde herramientas como Kallisto y Salmon se vuelven fundamentales.

En este artículo, exploraremos cómo Kallisto y Salmon, dos herramientas de cuantificación de transcripciones de última generación, pueden ser utilizadas de forma consistente y eficiente para integrar datos de secuenciación de doble cadena, garantizando la comparabilidad entre muestras y condiciones experimentales, y preparando el terreno para análisis integrados de expresión génica a gran escala.

Principios de cuantificación con Kallisto y Salmon

Algunas de las principales diferencias entre Kallisto y Salmon radican en su enfoque alineador y su algoritmo de cuantificación:

  • Kallisto: utiliza un algoritmo basado en pseudoalineación y indexación de k-mers. No requiere alineamiento real de reads, lo que lo hace extremadamente rápido. Genera un archivo de cuantificación en formato abundance.h5 o abundance.tsv.
  • Salmon: también emplea pseudoalineación, pero con un modelo de probabilidad más sofisticado y un enfoque más flexible. Permite el uso de quasi-mapping y ofrece un mayor control sobre el ajuste del modelo de expresión. Sus resultados se guardan en quant.sf o quant.genes.sf.

Ambas herramientas pueden procesar datos de secuenciación de doble cadena directamente, siempre que se proporcionen los archivos de entrada adecuados (como reads_1.fastq y reads_2.fastq) y se establezcan los parámetros correctos.

Preparación de datos para la integración

Antes de la cuantificación, es crucial asegurarse de que los datos de secuenciación de doble cadena estén correctamente preprocesados. Esto incluye:

  • Control de calidad con fastqc y multiqc.
  • Eliminación de adaptadores con cutadapt o Trimmomatic.
  • Verificación de la longitud y la distribución de los reads.

Una vez limpios, los archivos FASTQ deben ser indexados con Kallisto o Salmon usando la transcripción de referencia (por ejemplo, transcripts.fasta).

Ejemplo: Indexación con Kallisto

kallisto index -i transcripts.idx transcripts.fasta

Ejemplo: Indexación con Salmon

salmon index -i transcripts_index -t transcripts.fasta --gencode

Cuantificación de datos de secuenciación de doble cadena

La cuantificación debe realizarse de forma consistente para todas las muestras del estudio. Ambas herramientas aceptan datos de secuenciación de doble cadena mediante el uso de pares de archivos FASTQ.

Cuantificación con Kallisto

kallisto quant -i transcripts.idx -o sample1_kallisto \ 
    --threads 8 reads_1.fastq.gz reads_2.fastq.gz

Cuantificación con Salmon

salmon quant -i transcripts_index -o sample1_salmon \ 
    -p 8 --validateMappings --seqBias \ 
    -1 reads_1.fastq.gz -2 reads_2.fastq.gz

Nota: El parámetro --seqBias es recomendado en datos de secuenciación de doble cadena para corregir sesgos en la secuenciación. El uso de --validateMappings permite una estimación más precisa de la expresión.

Integración de resultados de cuantificación

Una vez obtenidos los archivos de cuantificación (como abundance.tsv de Kallisto o quant.sf de Salmon), el siguiente paso es integrarlos en un único objeto de expresión para análisis posterior. Esto se puede hacer con herramientas como R o Python.

Ejemplo en R usando tximport

library(tximport)
library(dplyr)

# Ruta a los directorios de cuantificación
files <- list.files("./quant", pattern = "quant.sf", full.names = TRUE)

# Importar datos
txi <- tximport(files, type = "salmon", txOut = TRUE)

# Ver resultados
head(txi$counts)

Ejemplo en Python usando pandas

import pandas as pd

# Cargar datos desde archivos Salmon
files = ["sample1_salmon/quant.sf", "sample2_salmon/quant.sf"]

# Leer y unir datos
dfs = []
for f in files:
    df = pd.read_csv(f, sep='\t')
    df = df["Name"]  # gene_id
    df["count"] = df["NumReads"]  # total read counts
    dfs.append(df)

# Unir todos los datos
combined = pd.concat(dfs, axis=1)
combined.columns = ["gene_id", "sample1", "sample2"]

Resumen

Kallisto y Salmon ofrecen soluciones rápidas, precisas y altamente escalables para la cuantificación de transcripciones en datos de secuenciación de doble cadena. Su capacidad para manejar pares de reads de forma eficiente permite la integración robusta de datos entre múltiples muestras. Al seguir buenas prácticas en el preprocesamiento, indexación y cuantificación, y al utilizar herramientas como tximport o pandas para la integración, los investigadores pueden construir matrices de expresión de alta calidad que son esenciales para análisis posteriores de expresión diferencial, integración de datos y descubrimiento de patrones biológicos.

Errores comunes / Buenas prácticas

  • ❌ No usar --seqBias en Salmon con datos de secuenciación de doble cadena: puede introducir sesgos sistemáticos.
  • ✅ Asegurarse de que el índice (Kallisto/Salmon) esté generado con el mismo archivo de transcripción que se usará en todos los análisis.
  • ❌ No validar el formato de los archivos de cuantificación antes de integrarlos.
  • ✅ Usar tximport en R o scripts de Python para asegurar que todos los genes estén alineados correctamente.
  • ✅ Documentar todos los parámetros usados en cuantificación para reproducibilidad.
