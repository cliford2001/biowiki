Detección de variabilidad estructural con LUMPY

Introducción: ¿Qué es la variabilidad estructural y por qué importa?

La variabilidad estructural (SV, por sus siglas en inglés) se refiere a alteraciones en el genoma de gran tamaño que incluyen deletiones, duplicaciones, inserciones, inversiones y translocaciones. A diferencia de los variantes de un solo nucleótido (SNVs) o los pequeños indels, las SVs pueden afectar múltiples genes, promotores, o regiones reguladoras, y están fuertemente asociadas con enfermedades genéticas, cáncer y variabilidad fenotípica. Detectar estas variantes de forma precisa y completa es un desafío clave en genómica moderna.

En este artículo, exploramos LUMPY (LIneUP for Mapping and genotyping of structural Variants), una herramienta de detección de SVs diseñada para análisis de secuenciación de próxima generación (NGS). LUMPY se distingue por su capacidad de integrar múltiples tipos de señales de secuenciación —como split reads, discordant pairs, read depth y soft-clipped reads— en un modelo estadístico unificado para identificar SVs de forma robusta, especialmente en datos de exoma o secuenciación del genoma completo (WGS).

Desarrollo: Cómo funciona LUMPY

LUMPY opera en dos fases principales: la generación de señales y el modelo de detección.

1. Preparación del alineamiento

Antes de ejecutar LUMPY, es esencial tener un alineamiento de alta calidad de los reads a un genoma de referencia. Las herramientas más comunes son STAR (para RNA-seq) o BWA-MEM (para DNA-seq). El archivo de salida debe estar en formato BAM, ordenado y indexado con samtools:

 samtools sort alineamiento.unsorted.bam -o alineamiento.sorted.bam
 samtools index alineamiento.sorted.bam

2. Generación de señales con LUMPY

LUMPY utiliza una combinación de cuatro tipos de evidencia:

  • Split reads: reads que se alinean a dos regiones distintas del genoma.
  • Discordant pairs: pares de reads que se alinean con distancia o orientación inesperada.
  • Read depth: variaciones en la cobertura de secuenciación que indican duplicaciones o eliminaciones.
  • Soft-clipped reads: reads con extremos truncados, común en translocaciones o grandes inserciones.

Para extraer estas señales, se usa el comando lumpy en modo make:

 lumpy -X 10000 -d 50000 -f 2 -l 100000 -o lumpy.bedpe alineamiento.sorted.bam

Donde los parámetros significan:

  • -X: tamaño máximo de una SV (en bp).
  • -d: desvío máximo permitido para pares discordantes.
  • -f: factor de penalización para read depth.
  • -l: longitud mínima de fragmento para considerar.
  • -o: nombre del archivo de salida en formato BEDPE.

3. Genotipado y filtrado

Una vez generada la lista de posibles SVs, se realiza el genotipado para estimar la probabilidad de que un individuo porte una variante. Esto se hace con el comando lumpy en modo genotyper:

 lumpy -X 10000 -d 50000 -f 2 -l 100000 -o genotipos.vcf -c 10000 -i 100000 -t 100000 alineamiento.sorted.bam lumpy.bedpe

El archivo de salida es un VCF que contiene información de genotipo, calidad y frecuencia de la variante.

Resumen

LUMPY es una herramienta poderosa y flexible para la detección de variabilidad estructural, especialmente útil en estudios de genómica humana y canceres. Su capacidad para integrar múltiples tipos de evidencia de NGS hace que sea más sensible que métodos basados en un solo tipo de señal. Sin embargo, requiere datos de alineamiento de alta calidad y configuración cuidadosa de parámetros para evitar falsos positivos.

La detección de SVs no es solo un paso técnico: es una clave para descubrir las bases genéticas de enfermedades complejas. LUMPY, junto con herramientas como SVIM, GridSS o DeepVariant, forma parte del arsenal indispensable para los bioinformáticos modernos.

Errores comunes / buenas prácticas

  • Usar datos de alineamiento sin ordenar o indexar: LUMPY fallará si el archivo BAM no está indexado.
  • Configurar parámetros demasiado laxos: esto genera muchos falsos positivos. Siempre validar con herramientas de supervisión como DeepVariant o Genome-Assembly-Validator.
  • No filtrar resultados por calidad: usar vcftools o bcftools para eliminar variantes de baja confianza.
  • Ignorar el contexto biológico: no toda SV detectada es relevante. Siempre anotar con herramientas como ANNOVAR o VEP.
  • Usar LUMPY en datos de RNA-seq sin ajustar: LUMPY está optimizado para DNA-seq. Para RNA-seq, considera herramientas como STAR-Fusion o SQANTI3.
