Análisis de expresión temporal con scVelo y RNA velocity

Introducción: Comprender el flujo de expresión génica en células individuales

El análisis de la expresión temporal en datos de secuenciación de RNA de células individuales (scRNA-seq) permite reconstruir trayectorias de diferenciación celular, transiciones de estado y dinámicas de expresión génica a nivel de tiempo. Aunque las técnicas tradicionales de agrupamiento y reducción de dimensionalidad (como UMAP o t-SNE) ofrecen una visión estática del espacio celular, no capturan el orden temporal ni las direcciones de cambio. En este contexto, scVelo y RNA velocity emergen como herramientas clave para inferir el estado futuro de las células a partir de la proporción de transcripciones recién sintetizadas (unspliced) y maduras (spliced).

RNA velocity se basa en el principio de que las células en proceso de activación génica presentan una mayor proporción de mRNA no procesado (unspliced) respecto a su correspondiente mRNA procesado (spliced). Esta diferencia temporal en la acumulación de transcripciones permite estimar la dirección y velocidad de los cambios en la expresión génica. scVelo implementa este cálculo y lo integra con modelos de regresión y grafos de transición para generar un mapa de dirección celular dinámico.

Desarrollo: De los datos a la inferencia de trayectorias

Preprocesamiento de datos

El proceso comienza con datos de scRNA-seq en formato h5ad (de Scanpy) o loom. Es fundamental realizar un preprocesamiento riguroso: filtrado de células y genes por calidad, normalización (por ejemplo, con scanpy.pp.normalize_total), y transformación logarítmica. scVelo requiere matrices de contadores de genes para transcripciones spliced y unspliced, que suelen generarse durante el alineamiento con herramientas como STAR o kallisto con el modo --quant y --include-intron-exons.

Estimación de RNA velocity

Una vez preparados los datos, se utiliza scv.tl.velocity para calcular la velocidad de expresión. Este paso estima la tasa de producción y degradación de mRNA y genera un vector de velocidad por célula. El código de ejemplo en Python es:

import scanpy as sc
import scvelo as scv

# Cargar datos
adata = sc.read_h5ad("data/filtered_gene_bc_matrices.h5ad")

# Preprocesar
scv.pp.normalize_per_cell(adata)
scv.pp.log1p(adata)

# Calcular velocidad
scv.tl.velocity(adata, n_jobs=8)

Este paso puede requerir ajustes si los datos no contienen información sobre transcripciones spliced/unspliced. En caso contrario, se debe usar scv.tl.velocity_graph para construir un grafo de transición basado en la velocidad.

Reconstrucción de trayectorias celulares

scVelo permite reconstruir trayectorias mediante la inferencia de un grafo de transición. Se aplica scv.tl.recover_dynamics para ajustar un modelo de dinámica genética por célula, lo que permite predecir el estado futuro de cada célula. Este modelo se basa en ecuaciones diferenciales que relacionan la velocidad con la expresión actual.

scv.tl.recover_dynamics(adata, fit_scatter=True)
scv.tl.velocity_graph(adata)

El grafo resultante puede visualizarse con scv.pl.velocity o scv.pl.velocity_graph, mostrando direcciones de cambio y conexiones entre estados celulares.

Resumen

El análisis de expresión temporal con scVelo y RNA velocity permite ir más allá de la representación estática de células, ofreciendo un mapa dinámico de diferenciación y transición de estados. Al combinar información de transcripciones recién sintetizadas y maduras, se pueden inferir direcciones de cambio, predecir estados futuros y validar hipótesis sobre los mecanismos reguladores del desarrollo celular. Esta aproximación es esencial en estudios de biología del desarrollo, regeneración, y enfermedad, donde el orden temporal es clave para comprender la función celular.

Errores comunes / buenas prácticas

  • Usar datos sin información de spliced/unspliced: scVelo requiere matrices separadas. Si no se tiene, no se puede aplicar RNA velocity.
  • No normalizar adecuadamente: El uso de raw counts puede sesgar la estimación de velocidad. Siempre normalizar antes de calcular.
  • Ignorar la calidad de los datos: Células con baja cobertura o alta fracción de genes mitocondriales deben filtrarse.
  • Aplicar recover_dynamics sin verificar la calidad del grafo: Es recomendable revisar el resultado con scv.pl.velocity_graph para asegurarse de que las direcciones son coherentes.
  • Interpretar el grafo como un árbol estricto: Aunque puede sugerir trayectorias, el grafo puede contener ciclos o bifurcaciones, lo que refleja la heterogeneidad biológica real.

En resumen, scVelo es una herramienta poderosa, pero requiere cuidado en el preprocesamiento y una interpretación crítica de los resultados. Su uso combinado con otros métodos de análisis (como pseudotiempos de Seurat o Pseudobulk) mejora la robustez de las inferencias.
