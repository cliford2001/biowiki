Validación de variantes con GATK VariantFiltration y recalibración de calidad

Introducción: Por qué validar variantes es clave en bioinformática

La detección de variantes genómicas (SNVs, indels, etc.) es un pilar fundamental en estudios de genómica médica, evolución y enfermedades hereditarias. Sin embargo, no todas las variantes llamadas por los algoritmos de llamado (como GATK HaplotypeCaller) son biológicamente reales. Las llamadas falsas positivas pueden surgir debido a errores de secuenciación, artefactos de alineamiento, baja cobertura o variabilidad técnica. Por ello, es crucial aplicar filtros de calidad y métodos de recalibración para asegurar que los datos de variación sean fiables antes de su interpretación.

En este artículo, exploramos dos técnicas esenciales dentro del Genome Analysis Toolkit (GATK): VariantFiltration para aplicar reglas de filtro basadas en métricas de calidad, y VariantRecalibrator para generar modelos de calidad de variantes basados en datos reales del estudio. Juntas, estas herramientas permiten una validación robusta y automatizada de variantes.

Desarrollo: Técnicas clave para validación de variantes

1. Filtro de variantes con GATK VariantFiltration

VariantFiltration es una herramienta de GATK que permite marcar variantes como "filtered" basándose en criterios definidos por el usuario. Se aplica sobre un archivo VCF ya generado (por ejemplo, con HaplotypeCaller) y añade información de filtro en el campo FILTER del archivo.

Esta herramienta no elimina variantes; solo las etiqueta. El filtro real se realiza posteriormente con RemoveFilteredVariants si es necesario.

Ejemplo de uso de VariantFiltration:

gatk VariantFiltration \
  -V raw_variants.vcf.gz \
  -O filtered_variants.vcf.gz \
  --filter-expression "QD < 2.0" --filter-name "LowQD" \
  --filter-expression "FS > 60.0" --filter-name "HighFS" \
  --filter-expression "MQ < 40.0" --filter-name "LowMQ" \
  --filter-expression "MQRankSum < -12.5" --filter-name "LowMQRankSum" \
  --filter-expression "ReadPosRankSum < -8.0" --filter-name "LowReadPosRankSum"

En este ejemplo, se aplican filtros para variantes con baja calidad de profundidad (QD), alto sesgo de flujo (FS), baja calidad de alineamiento (MQ), y desviaciones en la distribución de posiciones de lecturas (MQRankSum, ReadPosRankSum). Estos valores son comunes en estudios de exoma y genoma completo.

2. Recalibración de calidad de variantes con VariantRecalibrator

Mientras que VariantFiltration aplica reglas fijas, VariantRecalibrator genera un modelo estadístico que asigna una calidad de llamado (como un valor de Phred score) a cada variante, basado en sus características observadas en datos reales.

El proceso se realiza en dos pasos:

  1. Se construye un modelo usando un conjunto de variantes de alta confianza (por ejemplo, de dbSNP).
  2. Se recalibran las variantes del estudio, obteniendo un VCF con puntuaciones de calidad ajustadas.

Paso 1: Generar el modelo de recalibración

gatk VariantRecalibrator \
  -V raw_variants.vcf.gz \
  -O recalibration_report.csv \
  --resource:snps,known=true,training=true,truth=true,prior=12.0 dbSNP.vcf.gz \
  --resource:indels,known=true,training=true,truth=true,prior=10.0 dbSNP.vcf.gz \
  -mode SNPs \
  -tranches 100.0,99.9,99.0,90.0

Este comando usa el archivo dbSNP.vcf.gz como fuente de variantes verdaderas para entrenar el modelo. El parámetro -mode especifica que se está recalibrando para SNVs. Las tranches (100, 99.9, etc.) definen los umbrales de confianza para la posterior filtración.

Paso 2: Aplicar el modelo a las variantes

gatk ApplyVQSR \
  -V raw_variants.vcf.gz \
  -O recalibrated_variants.vcf.gz \
  --recal-file recalibration_report.csv \
  --tranches-file recalibration_tranches.txt \
  -mode SNPs

Este paso genera un VCF con variantes recalibradas, donde cada variante tiene una puntuación de calidad ajustada. Las variantes con puntuación baja pueden ser filtradas en un paso posterior.

Resumen

La validación de variantes es un paso crítico en el análisis genómico. VariantFiltration permite aplicar reglas basadas en métricas de calidad (como QD, FS, MQ) para etiquetar variantes sospechosas. Por otro lado, VariantRecalibrator y ApplyVQSR ofrecen un enfoque más sofisticado mediante modelos estadísticos que ajustan la calidad de las llamadas basándose en datos de referencia de alta confianza. Juntos, estos métodos garantizan que solo las variantes más fiables sean consideradas en análisis posteriores, mejorando la precisión y reproducibilidad de los resultados.

Errores comunes / buenas prácticas

  • Usar valores de corte arbitrarios: Evita usar valores de filtro como FS > 60 sin validarlos en tu tipo de datos. Los umbrales pueden variar según el estudio (exoma vs. genoma completo).
  • No usar datos de referencia: Es esencial proporcionar un conjunto de variantes verdaderas (como dbSNP) para entrenar el modelo de recalibración. Sin datos de verdad, el modelo no es fiable.
  • Ignorar los archivos de tranches: El archivo recalibration_tranches.txt es clave para filtrar variantes por nivel de confianza. Usa ApplyVQSR con el parámetro --tranches-file para aplicar filtros basados en estos niveles.
  • Aplicar filtrado antes de recalibrar: No filtres variantes antes de usar VariantRecalibrator. El modelo necesita todos los datos (incluso los de baja calidad) para aprender correctamente.
  • Verificar el resultado con VCFtools: Usa vcftools --filter-stats para analizar el impacto del filtro y la distribución de métricas después del procesamiento.
