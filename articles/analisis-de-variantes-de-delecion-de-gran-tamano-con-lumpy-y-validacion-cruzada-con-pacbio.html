Análisis de variantes de deleción de gran tamaño con LUMPY y validación cruzada con PacBio

Introducción: Por qué las deleciones de gran tamaño importan

Las deleciones estructurales de gran tamaño (genómicas) son eventos variantes que implican la pérdida de segmentos de ADN de más de 50 kilobases (kb), y desempeñan un papel crucial en la evolución, la patogénesis y la variabilidad fenotípica entre individuos. A diferencia de variantes de un solo nucleótido (SNVs) o inserciones/deleciones pequeñas (indels), las deleciones de gran tamaño pueden afectar múltiples genes, reguladores de genes o regiones de control, generando fenotipos complejos o enfermedades genéticas.

La detección precisa de estas variantes es un desafío técnico debido a la limitada capacidad de secuenciación de corta lectura (short-read sequencing), como las plataformas Illumina, que no pueden cubrir largas regiones de repetición o estructuras complejas. En este contexto, herramientas como LUMPY (LIgand-based Unification of Multiple k-mer-based approaches) emergen como soluciones clave, combinando múltiples tipos de evidencia de secuenciación (split reads, discordant pairs, read depth) para identificar deleciones estructurales con alta sensibilidad y especificidad.

Enfoque con LUMPY: integración de señales de secuenciación de corta lectura

LUMPY es una herramienta de análisis de variantes estructurales (SVs) desarrollada para detectar eventos como deleciones, duplicaciones, translocaciones y inversiones. Su fortaleza radica en su capacidad de integrar múltiples tipos de señal desde datos de secuenciación de corta lectura (WGS o WES).

El algoritmo utiliza una estrategia bayesiana para combinar evidencias de:

  • Split reads: fragmentos de ADN que se alinean a dos regiones genómicas distintas, indicando un punto de ruptura.
  • Discordant read pairs: pares de lecturas que se alinean con un tamaño de inserto mucho mayor (o menor) del esperado, sugiriendo una deleción (o duplicación).
  • Read depth: cambios en la profundidad de cobertura que pueden indicar la pérdida de una región genómica.

Para ejecutar LUMPY, se requiere un archivo de alineación BAM (obtenido con herramientas como bowtie2, STAR o BWA-MEM) y un archivo de índice de referencia genómica (FASTA + índices). El flujo típico es:

lumpy -B input.bam -S split_reads.txt -D discordant_pairs.txt -R ref.fasta -o variants.vcf

Donde split_reads.txt y discordant_pairs.txt son archivos generados previamente con lumpyexpress (una interfaz simplificada).

Validación cruzada con PacBio: el oro del oro para SVs

Si bien LUMPY es muy útil, sus resultados deben ser validados, especialmente cuando se trata de deleciones de gran tamaño. Las lecturas de larga duración (long-read sequencing) de plataformas como PacBio (HiFi) ofrecen una ventaja insuperable: pueden cubrir regiones completas de ADN, incluyendo repeticiones, estructuras secundarias y secuencias de alta complejidad, lo que permite visualizar directamente las rupturas de los eventos estructurales.

La validación cruzada con PacBio implica:

  1. Secuenciación del mismo individuo con PacBio HiFi (long-reads).
  2. Alineación de las lecturas de PacBio a la referencia con PBMM2 o minimap2.
  3. Identificación de variantes estructurales usando Sniffles o PB-SC.
  4. Comparación de las regiones de deleción detectadas por LUMPY con las de PacBio mediante herramientas como bcftools o pyvcf, buscando concordancia en la posición, tamaño y extremos.

Este enfoque permite distinguir verdaderas deleciones de falsos positivos, como artefactos de alineación o regiones con baja cobertura en secuenciación de corta lectura.

Resumen

El análisis de variantes de deleción de gran tamaño es un campo crítico en genómica funcional y clínica. LUMPY se posiciona como una herramienta poderosa para detectar estas variantes mediante la integración de múltiples señales de secuenciación de corta lectura. Sin embargo, su eficacia depende de una validación rigurosa, donde las tecnologías de lectura larga, como PacBio HiFi, proporcionan una prueba de oro para confirmar la existencia y exactitud de las deleciones detectadas.

El enfoque combinado de LUMPY + PacBio no solo mejora la fiabilidad del análisis, sino que también permite una caracterización precisa de los extremos de ruptura, esenciales para entender mecanismos moleculares y desarrollar terapias dirigidas.

Errores comunes / buenas prácticas

  • Usar LUMPY sin validación: Nunca confíes en un resultado de LUMPY como oro en bruto. Siempre verifica con datos de lectura larga o técnicas de PCR.
  • Ignorar el tamaño de deleción: LUMPY puede detectar deleciones pequeñas, pero su sensibilidad cae en regiones con baja cobertura. Asegúrate de filtrar por tamaño mínimo (por ejemplo, >50 kb).
  • Usar datos de WES en lugar de WGS: La secuenciación de exomas (WES) no cubre suficientemente la región genómica completa. Usa WGS para detectar SVs de gran tamaño.
  • Procesar datos de PacBio sin ajustar parámetros: Herramientas como Sniffles requieren ajustes específicos para datos de PacBio HiFi (por ejemplo, --minimap2-params).
  • Comparar directamente archivos VCF sin normalización: Usa herramientas como vcflib o vcfconsensus para normalizar variantes antes de comparar.
