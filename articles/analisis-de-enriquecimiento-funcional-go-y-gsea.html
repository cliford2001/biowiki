Análisis de enriquecimiento funcional GO y GSEA

Introducción: ¿Qué es el enriquecimiento funcional?

El análisis de enriquecimiento funcional (Functional Enrichment Analysis) es una técnica fundamental en bioinformática que permite interpretar listas de genes diferencialmente expresados en términos biológicos. Tras un experimento de secuenciación de RNA (RNA-seq) o un análisis de microarreglos, los investigadores obtienen listas de genes que se han regulado significativamente. Sin embargo, estos resultados no revelan directamente su función biológica, por lo que se requiere una forma de interpretarlos a nivel de vía, proceso biológico o componente celular.

Los dos métodos más utilizados para este fin son el Análisis de enriquecimiento de Gene Ontology (GO) y el Gene Set Enrichment Analysis (GSEA). Aunque ambos buscan responder preguntas similares, se diferencian en su enfoque, supuestos y aplicaciones. Este artículo explica sus fundamentos, diferencias clave, uso práctico en R y Python, y las mejores prácticas para evitar errores comunes.

Gene Ontology (GO): análisis basado en conjuntos de genes

GO es una base de conocimiento estructurada que describe la función de los genes en tres categorías:

  • Proceso biológico (Biological Process): una serie de eventos que contribuyen a un fenómeno biológico, como la replicación del ADN o la respuesta inmune.
  • Componente celular (Cellular Component): estructuras o localizaciones dentro de la célula, como el núcleo, la membrana plasmática o el ribosoma.
  • Actividad molecular (Molecular Function): la función molecular específica que realiza una proteína, como la unión de ADN o la transferencia de electrones.

El análisis de enriquecimiento GO compara una lista de genes de interés (por ejemplo, genes diferencialmente expresados) con una base de datos de genes asociados a términos GO. El objetivo es identificar si ciertos términos GO están sobrerepresentados en la lista de genes, lo que indica que esos procesos están significativamente involucrados en el fenómeno estudiado.

Ejemplo de uso en R con clusterProfiler


library(clusterProfiler)
library(org.Hs.eg.db)

# Supongamos que `upregulated_genes` es una lista de genes con p-valor < 0.05
upregulated_genes <- c("TP53", "CDK2", "BCL2", "MYC", "CASP3")

# Convertir IDs de genes a símbolos de organismos (ej. humano)
up_genes <- bitr(upregulated_genes, fromType = "SYMBOL", toType = "ENTREZID", OrgDb = org.Hs.eg.db)

# Realizar análisis de enriquecimiento GO
ego <- enrichGO(gene = up_genes$ENTREZID,
                universe = keys(org.Hs.eg.db),
                OrgDb = org.Hs.eg.db,
                ont = "BP", # o "CC" o "MF"
                pAdjustMethod = "BH")

# Ver resultados
head(ego@result)

# Visualizar resultados
barplot(ego, showCategory = 10)

Gene Set Enrichment Analysis (GSEA): análisis basado en rango de expresión

A diferencia de GO, que opera sobre listas binarias de genes (incluso/no incluido), GSEA utiliza el ranking de todos los genes en función de su nivel de expresión (por ejemplo, log2FC o p-valor). Este enfoque permite detectar cambios sutiles en vías que pueden no alcanzar un umbral de significancia individual, pero que se manifiestan colectivamente.

El algoritmo de GSEA calcula una estadística de enriquecimiento (ES) para cada vía (gene set), que mide cuán fuerte está el rango de expresión en una vía en comparación con el resto. Si el rango está concentrado en los genes de la vía (es decir, todos los genes altamente expresados están en esa vía), se considera enriquecido. Luego se evalúa la significancia estadística mediante permutaciones.

Ejemplo de uso en Python con GSEApy


import gseapy as gp

# Supongamos que `gene_scores` es un diccionario con genes y sus valores de log2FC
# Ejemplo: {'TP53': 2.1, 'CDK2': 1.8, 'BCL2': -0.5, 'MYC': 3.0, 'CASP3': 1.2}

# Cargar una colección de vías (por ejemplo, MSigDB)
gene_sets = gp.get_gmt('c2.cgp.v7.5.1.symbols.gmt')

# Realizar GSEA
results = gp.gsea(
    data=gene_scores,
    gene_sets=gene_sets,
    nperm=1000,
    outdir='./gsea_results',
    min_size=15,
    max_size=500,
    method='signal2noise',
    verbose=True
)

# Obtener resultados
results['gsea_results'].head()

Comparación y elección del método

La elección entre GO y GSEA depende del tipo de pregunta que se quiere responder:

  • Usa GO cuando tienes una lista clara de genes diferencialmente expresados (por ejemplo, tras aplicar un umbral de p-valor) y deseas identificar vías biológicas específicas asociadas a ellos.
  • Usa GSEA cuando sospechas que una vía puede estar afectada de forma sutil, pero no todos sus genes son significativos individualmente, o cuando el análisis se basa en un ranking continuo de expresión.

Además, GSEA es más robusto frente al ruido y permite detectar vías con efectos combinados. Sin embargo, requiere más tiempo computacional y una base de datos más rica (como MSigDB).

Resumen

El análisis de enriquecimiento funcional es una pieza clave en la interpretación de datos de alto rendimiento. El análisis GO es ideal para identificar vías biológicas significativamente representadas en listas de genes diferencialmente expresados, mientras que el GSEA es más sensible a cambios sutiles y colectivos en vías. Ambos métodos, aunque distintos en su enfoque, complementan al investigador al ofrecer una visión biológica profunda más allá de los números.

Errores comunes y buenas prácticas

  • No usar una base de datos actualizada: siempre verifica que estés usando una versión reciente de GO o MSigDB. Las bases de datos evolucionan constantemente.
  • Ignorar el ajuste de múltiples pruebas: siempre aplica métodos como BH (Benjamini-Hochberg) para controlar el FDR (False Discovery Rate).
  • Tratar todos los genes como iguales: en GSEA, el ranking es clave. Asegúrate de que la métrica de ranking (p-valor, log2FC) sea adecuada para tu pregunta.
  • Interpretar resultados sin validación biológica: los resultados estadísticos deben ser combinados con conocimiento biológico y pruebas experimentales.
  • Olvidar el tamaño de las vías: vías muy grandes pueden tener más genes, lo que aumenta el riesgo de enriquecimiento falso. Usa filtros de tamaño (por ejemplo, entre 15 y 500 genes).
