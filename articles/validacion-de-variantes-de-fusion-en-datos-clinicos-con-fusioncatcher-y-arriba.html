Validación de variantes de fusión en datos clínicos con FusionCatcher y Arriba

Introducción: Fusión de genes en medicina clínica

Las fusiones génicas son eventos genéticos que ocurren cuando dos genes distintos se unen, produciendo un nuevo gen quimérico con funciones potencialmente oncogénicas. Estas variantes son altamente relevantes en oncología, especialmente en tumores como el sarcoma de Ewing, leucemia mieloide aguda y cáncer de pulmón no pequeño. Su detección precisa y validación rigurosa es fundamental para el diagnóstico, pronóstico y selección de terapias dirigidas.

Dado su impacto clínico, se requieren métodos bioinformáticos robustos capaces de identificar fusiones a partir de secuenciación de RNA (RNA-seq). En este artículo, exploramos el uso combinado de FusionCatcher y Arriba para validar variantes de fusión en datos clínicos, destacando sus fortalezas, limitaciones y mejores prácticas para asegurar resultados fiables.

Desarrollo

1. Flujo de trabajo: de los datos crudos a la fusión validada

El proceso de validación de fusiones en datos clínicos requiere un pipeline integrado que combine detección, filtrado y verificación. FusionCatcher y Arriba se complementan perfectamente en este flujo:

  • FusionCatcher: Es un herramienta de detección de fusiones altamente sensible, que utiliza alineamientos de reads contra un genoma de referencia (por ejemplo, GRCh38) y analiza los patrones de alineamiento atípicos para identificar posibles fusiones.
  • Arriba: Es una herramienta especializada en validar fusiones detectadas por otros pipelines, centrada en el análisis de reads de secuenciación que cruzan el punto de fusión. Arriba evalúa la calidad del soporte de datos y proporciona una puntuación de confianza para cada variante.

2. Ejecución de FusionCatcher: detección de fusiones potenciales

El primer paso consiste en ejecutar FusionCatcher sobre los datos alineados (por ejemplo, archivos BAM generados con STAR o HISAT2). El siguiente comando es un ejemplo típico:

fusioncatcher -i aligned_reads.bam \
              -r /path/to/human_genome_reference \
              -o fusion_results \
              -t 8 \
              --no-soft-clip

Este comando genera varios archivos de salida, incluyendo:

  • fusion_results.fusion_predictions.txt: lista de fusiones detectadas con información de soporte, tipo de fusión y puntuación de confianza.
  • fusion_results.fusion_genes.txt: resumen de genes implicados en fusiones.
  • Archivos HTML para visualización.

3. Validación con Arriba: evaluación de la evidencia de fusión

Con las fusiones detectadas por FusionCatcher, se utiliza Arriba para realizar una validación más rigurosa. Arriba requiere un archivo BAM con reads alineados y el archivo de fusiones en formato de lista de genes (por ejemplo, desde FusionCatcher).

El siguiente comando muestra cómo usar Arriba:

arriba --input aligned_reads.bam \
       --fusion-list fusion_results.fusion_predictions.txt \
       --genome hg38 \
       --output arriba_validation \
       --threads 8

Arriba genera un reporte detallado que incluye:

  • El número de reads que cruzan el punto de fusión (fusion-supporting reads).
  • La calidad de los alineamientos en los extremos de la fusión.
  • Una puntuación de confianza que combina múltiples métricas.
  • Posibles falsos positivos basados en artefactos de alineamiento.

Resumen

La validación de variantes de fusión en datos clínicos requiere una estrategia multi-etapa que combine detección sensible con validación rigurosa. FusionCatcher ofrece una detección robusta de fusiones potenciales a partir de datos de RNA-seq, mientras que Arriba proporciona una evaluación de calidad detallada del soporte de reads para cada fusión candidata. Su uso conjunto permite reducir el número de falsos positivos y aumentar la confianza en los resultados, lo cual es esencial en un contexto clínico donde decisiones terapéuticas dependen de estos hallazgos.

El flujo de trabajo debe incluir preprocesamiento cuidadoso, validación de calidad de datos, y revisión manual de variantes con alto nivel de confianza. Además, es fundamental mantener un registro claro de los parámetros utilizados y los resultados generados para reproducibilidad y auditoría clínica.

Errores comunes / buenas prácticas

  • Usar referencias genómicas desactualizadas: Asegúrese de utilizar un genoma de referencia actualizado (GRCh38) y una base de datos de genes actualizada para evitar falsos negativos o falsos positivos.
  • Ignorar filtros de calidad: Siempre revisar los resultados de Arriba y descartar fusiones con bajo soporte de reads (menos de 3-5 reads cruzando el punto de fusión) o con alineamientos de baja calidad.
  • Confundir detección con validación: FusionCatcher es solo una herramienta de detección. Nunca use sus resultados directamente en informes clínicos sin validación adicional con Arriba o herramientas similares.
  • No documentar el pipeline: Documente cada paso, parámetros y versiones de herramientas para cumplir con estándares clínicos y regulaciones (como CLIA o ISO 15189).
  • Usar datos de baja calidad: Evite datos con bajo rendimiento de secuenciación, alta tasa de contaminación o mala calidad de los reads. Siempre validad la calidad del RNA previa a la secuenciación.
