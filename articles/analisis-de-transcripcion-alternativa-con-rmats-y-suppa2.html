Análisis de transcripción alternativa con rMATS y SUPPA2

¿Qué es la transcripción alternativa y por qué es importante?

La transcripción alternativa es un proceso clave en la regulación génica que permite que un solo gen produzca múltiples isoformas de ARN mensajero (ARNm) mediante diferentes combinaciones de exones. Este fenómeno amplía significativamente la diversidad proteica sin incrementar el número de genes, y está implicado en procesos fisiológicos normales así como en enfermedades como el cáncer, trastornos neurológicos y trastornos del desarrollo.

El análisis de la transcripción alternativa permite identificar cambios en la expresión de isoformas entre condiciones biológicas (por ejemplo, tumor vs. tejido normal), lo que puede revelar nuevos mecanismos patológicos, biomarcadores de diagnóstico o dianas terapéuticas. Herramientas como rMATS y SUPPA2 son dos de las más utilizadas en el análisis de datos de secuenciación de ARN (RNA-seq) para detectar y cuantificar eventos de splicing alternativo.

Enfoques comparativos: rMATS vs SUPPA2

En el análisis de transcripción alternativa, dos herramientas destacan por su robustez y uso amplio: rMATS (replicate Multivariate Analysis of Transcript Splicing) y SUPPA2 (Splicing Analysis Pipeline Package 2).

rMATS: análisis de eventos de splicing con métodos estadísticos robustos

rMATS es una herramienta diseñada para detectar eventos de splicing alternativo con base en datos de RNA-seq con replicados biológicos. Utiliza un modelo estadístico bayesiano para evaluar diferencias significativas en la proporción de splicing entre condiciones, y es especialmente útil para identificar eventos como:

  • Skipping Exon (SE): Exon que se salta en una isoforma.
  • Alternative 5' Splice Site (A5SS): Uso de un sitio de splicing 5' distinto.
  • Alternative 3' Splice Site (A3SS): Uso de un sitio de splicing 3' distinto.
  • Mutually Exclusive Exons (MXE): Dos exones que no se incluyen juntos.
  • Retained Intron (RI): Intrón que se mantiene en el ARNm maduro.

rMATS requiere archivos de alineamiento en formato BAM como entrada, y es capaz de manejar múltiples condiciones y replicados. Su salida incluye un archivo de resultados con estadísticas de significancia (p-valor ajustado) y la dirección del cambio (up o down) en la proporción de splicing.

SUPPA2: integración de análisis, cuantificación y visualización

SUPPA2 es una plataforma más integral que no solo detecta eventos de splicing alternativo, sino que también permite la cuantificación de isoformas, la integración con datos de expresión génica y la visualización de resultados. A diferencia de rMATS, SUPPA2 puede trabajar directamente con archivos de expresión (como los generados por Salmon o kallisto) y ofrece un flujo de trabajo más completo:

  • Deteción de eventos de splicing alternativo en múltiples condiciones.
  • Cuantificación de isoformas a partir de datos de expresión.
  • Identificación de patrones de splicing diferencial.
  • Generación de gráficos y análisis de enriquecimiento funcional.

Supplementalmente, SUPPA2 incluye un conjunto de herramientas para la visualización de resultados, como mapas de calor de cambios en el splicing y diagramas de red de eventos relacionados.

Flujo de trabajo típico: desde alineamiento hasta interpretación

El análisis de transcripción alternativa requiere una serie de pasos interconectados. A continuación se presenta un flujo de trabajo recomendado:

Paso 1: Alineamiento de reads

Los reads de RNA-seq deben alinearse a un genoma de referencia. Herramientas como STAR o Hisat2 son ampliamente utilizadas para este propósito. Ejemplo con STAR:

STAR \
  --runThreadN 8 \
  --genomeDir /path/to/genome \
  --readFilesIn sample1_R1.fastq sample1_R2.fastq \
  --outFileNamePrefix sample1_

Paso 2: Generación de archivos para rMATS

rMATS requiere archivos de alineamiento en formato BAM. Si se usan herramientas como Salmon o kallisto, se puede usar salmon quant con el parámetro --gencode para generar archivos de cuantificación que pueden ser usados en SUPPA2.

salmon quant \
  --threads 8 \
  --libType ISF \
  --gencode \
  --transcripts transcripts.fasta \
  --seqs reads.fastq \
  -o quant_output

Paso 3: Análisis con rMATS

Para ejecutar rMATS, se requiere un archivo de configuración con las rutas de los archivos BAM de cada condición. Ejemplo del comando:

rMATS --b1 condition1_rep1.bam condition1_rep2.bam \
      --b2 condition2_rep1.bam condition2_rep2.bam \
      --gtf annotation.gtf \
      --task splicing \
      --out rMATS_output \
      --nthread 8

La salida incluye un archivo summary.txt con eventos significativos y un archivo diffSplice.txt con los resultados de diferencias en splicing.

Paso 4: Análisis con SUPPA2

SUPPA2 puede procesar los resultados de rMATS o trabajar directamente con cuantificaciones. Ejemplo de análisis:

suppa.py --operation=summary --input=quant_output --output=suppa_summary
suppa.py --operation=splicing --input=quant_output --output=splicing_results

Esto genera resultados detallados de eventos de splicing con estadísticas adicionales y permite exportar datos para visualización.

Resumen

El análisis de la transcripción alternativa es una herramienta esencial para comprender la regulación post-transcripcional en biología funcional y patológica. rMATS ofrece un enfoque estadísticamente riguroso para detectar eventos de splicing diferencial entre condiciones, especialmente útil en estudios con replicados biológicos. SUPPA2, por su parte, proporciona una plataforma más completa que integra detección, cuantificación, visualización y análisis funcional, ideal para estudios exploratorios o integrativos.

La elección entre rMATS y SUPPA2 depende del objetivo del estudio, el tipo de datos disponibles y el nivel de integración deseado. En muchos casos, se recomienda usar rMATS para identificar eventos significativos y luego SUPPA2 para profundizar en la cuantificación y visualización.

Errores comunes y buenas prácticas

  • Usar archivos de alineamiento sin replicados: rMATS requiere replicados biológicos para una estimación robusta de la variabilidad. Sin replicados, los resultados pueden ser sesgados.
  • No validar el splicing con herramientas externas: Es recomendable cruzar resultados entre rMATS y SUPPA2, o con otras herramientas como MAJIQ o DEXSeq, para confirmar hallazgos.
  • Ignorar la calidad de los datos de entrada: Asegurarse de que los archivos BAM estén correctamente alineados y sin artefactos (usar herramientas como samtools y picard).
  • Usar un gtf incorrecto: El archivo de anotación debe coincidir con la versión del genoma utilizada durante el alineamiento.
  • Evaluar múltiples pruebas de significancia: Cuando se analizan muchos eventos, aplicar correcciones como Benjamini-Hochberg para controlar el FDR (tasa de descubrimiento falso).
