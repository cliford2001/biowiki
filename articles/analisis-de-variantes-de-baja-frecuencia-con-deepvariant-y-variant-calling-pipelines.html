Análisis de variantes de baja frecuencia con DeepVariant y variant calling pipelines

Introducción: ¿Qué son las variantes de baja frecuencia y por qué importan?

En bioinformática, las variantes de baja frecuencia (o low-frequency variants) se refieren a cambios genéticos que están presentes en una proporción de lecturas de secuenciación inferior al 10-20%, dependiendo del contexto biológico y técnico. A diferencia de las variantes de alta frecuencia (como los SNVs o indels en heterocigotos), estas variantes pueden ser subclonales, somáticas, o producto de errores de secuenciación, y su detección es crítica en estudios de cáncer, infecciones virales, microbiomas y trastornos hereditarios de aparición tardía.

La detección precisa de variantes de baja frecuencia es un desafío debido a la alta tasa de falsos positivos generada por errores de base, variabilidad técnica en la secuenciación (como los errores de amplificación), y la limitada profundidad de cobertura. Herramientas convencionales como bcftools call o GATK UnifiedGenotyper no están optimizadas para este escenario, ya que suelen aplicar umbral de confianza alto que filtra variantes reales con baja frecuencia. En este artículo, exploramos cómo DeepVariant y pipelines de variant calling especializados permiten una detección más sensible y específica de estas variantes.

DeepVariant: una alternativa basada en aprendizaje profundo

DeepVariant es una herramienta de llamado de variantes desarrollada por Google que utiliza redes neuronales profundas (CNN) para identificar SNVs e indels directamente desde los datos de alineamiento (BAM). A diferencia de los métodos tradicionales basados en reglas, DeepVariant aprende patrones de variación a partir de grandes conjuntos de datos de referencia, permitiendo una mayor sensibilidad y especificidad, especialmente en regiones difíciles (como repetitivas o con baja cobertura).

DeepVariant no se limita a variantes de alta frecuencia: al ajustar los umbrales de confianza y utilizar datos de secuenciación de alta profundidad (≥100x), puede identificar variantes de baja frecuencia con mayor precisión. Además, su modelo está entrenado para distinguir entre verdaderas variantes y artefactos técnicos, reduciendo significativamente el número de falsos positivos.

Flujo de trabajo con DeepVariant

El pipeline típico incluye:

  1. Alinamiento de lecturas con STAR o BWA-MEM.
  2. Preprocesamiento del BAM con samtools y picard para marcar duplicados y realizar ajustes de calidad.
  3. Generación de imágenes de entrada para DeepVariant usando deepvariant/make_examples (que crea una representación espacial de los alineamientos).
  4. Ejecución del modelo con deepvariant/call_variants.
  5. Postprocesamiento con deepvariant/postprocess_variants para filtrar por calidad y generar el VCF final.
python3 /path/to/deepvariant/make_examples \
  --mode calling \
  --ref hg38.fasta \
  --reads aligned_sorted.bam \
  --output_examples examples.tfrecord \
  --interval_list regions.interval_list
python3 /path/to/deepvariant/call_variants \
  --examples examples.tfrecord \
  --output_vcf output.vcf \
  --checkpoint_path /path/to/deepvariant/model.ckpt

Pipelines de variant calling especializados para variantes de baja frecuencia

Además de DeepVariant, existen pipelines específicos diseñados para el análisis de variantes subclonales y de baja frecuencia:

  • VarScan2: ofrece opciones para detección de variantes de baja frecuencia mediante el uso de varscan somatic o varscan mpileup2vcf con umbral de frecuencia ajustable (por ejemplo, --min-coverage 100 --min-allele-freq 0.05).
  • FreeBayes: es un llamador bayesiano que permite una alta sensibilidad. Puede configurarse para detectar variantes con frecuencias menores al 5% usando --min-coverage y --p-allele-freq.
  • LoFreq: diseñado específicamente para variantes de baja frecuencia. Su enfoque estadístico basado en la distribución de errores permite distinguir entre variantes reales y artefactos con alta precisión. Es ampliamente utilizado en estudios de cáncer y virus (como HIV o SARS-CoV-2).

Un enfoque recomendado es el uso de múltiples llamadores en paralelo seguido de un consenso (por ejemplo, con vcflib o bcftools), lo que mejora la confiabilidad de las variantes detectadas.

Resumen

El análisis de variantes de baja frecuencia requiere herramientas y pipelines que superen las limitaciones de los métodos tradicionales. DeepVariant, con su base en aprendizaje profundo, ofrece una alternativa altamente precisa para la detección de variantes, especialmente cuando se combina con datos de alta profundidad. Complementariamente, herramientas como LoFreq y FreeBayes están diseñadas explícitamente para este propósito, permitiendo la identificación de variantes subclonales en cáncer, infecciones virales, y microbiomas. El uso combinado de múltiples llamadores y la validación posterior son claves para garantizar la calidad de los resultados.

Errores comunes / buenas prácticas

  • Usar profundidad de cobertura insuficiente (menor a 100x) para detectar variantes de baja frecuencia: esto reduce la sensibilidad y aumenta el error de detección.
  • Ignorar el preprocesamiento de datos: no marcar duplicados, no realizar recalibración de calidad o no filtrar reads mal alineadas introduce artefactos que confunden los llamadores.
  • Aplicar umbral de frecuencia demasiado bajo sin validación: puede aumentar falsos positivos. Siempre validar con datos de control o con técnicas independientes (como Sanger).
  • Usar una sola herramienta: combinar múltiples llamadores (por ejemplo, DeepVariant + LoFreq) y usar consenso mejora la fiabilidad.
  • Descuidar la calidad de los datos de entrada: asegurarse de que el alineamiento sea correcto y que el BAM esté bien generado (con samtools index y picard ValidateSamFile).
