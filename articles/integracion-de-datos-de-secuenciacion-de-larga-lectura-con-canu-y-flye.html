Integración de datos de secuenciación de larga lectura con Canu y Flye

Introducción: Por qué integrar datos de larga lectura

La secuenciación de larga lectura (Long-Read Sequencing, LRS), con plataformas como Oxford Nanopore Technologies (ONT) y PacBio, ha revolucionado el ensamblaje genómico al resolver regiones complejas del ADN que los métodos de lectura corta no pueden abordar. Sin embargo, los datos de LRS suelen presentar una tasa de error más alta (hasta un 10-15% en ONT) en comparación con los datos de lectura corta (short-read). Para superar esta limitación, los investigadores recurren a la integración de datos: combinando la longitud de lectura con la alta precisión de los datos de secuenciación de corta lectura (short-read) para generar ensamblados de alta calidad.

Dos herramientas de software clave en este proceso son Canu y Flye. Ambas son herramientas de ensamblado de larga lectura de código abierto, diseñadas para manejar datos de ONT y PacBio con eficiencia y robustez. Aunque pueden funcionar independientemente, su verdadero potencial se despliega cuando se utilizan en conjunto con datos de lectura corta, permitiendo la corrección de errores y la mejora de la continuidad del ensamblado.

Conceptos clave: Canu y Flye

Canu: Ensamblado de larga lectura con corrección de errores

Canu (California University, Santa Cruz) es una herramienta robusta de ensamblado de larga lectura que combina alineamiento de lectura, corrección de errores y ensamblaje en un flujo de trabajo unificado. Su principal ventaja es su capacidad para manejar datos con alta tasa de error, utilizando un modelo estadístico para corregir errores en las lecturas antes de ensamblarlas. Canu puede utilizar datos de lectura corta para mejorar su corrección, lo que lo convierte en una opción ideal para integrar datos híbridos.

Flye: Ensamblado rápido y eficiente para datos de larga lectura

Flye (Fibonacci Long-Read Assembler) es otra herramienta de ensamblado de larga lectura, diseñada para ser rápida y eficiente, especialmente para datos de PacBio y ONT. Flye sigue un enfoque basado en grafos de de Bruijn, pero con optimizaciones que permiten manejar datos con alta variabilidad. Aunque su enfoque tradicional es el ensamblado de larga lectura autónomo, Flye también puede integrar datos de lectura corta para mejorar la corrección de errores y la calidad del ensamblado final.

Flujo de trabajo de integración: combinando Canu y Flye

El proceso de integración de datos de secuenciación de larga lectura con Canu y Flye se puede dividir en tres fases principales:

  1. Preprocesamiento: filtrado de datos de lectura corta y larga para eliminar artefactos, bajos rendimiento y contaminantes. Herramientas como fastp o Trimmomatic son comunes para este paso.
  2. Corrección de errores con datos híbridos: utilizar Canu o Flye para corregir errores en las lecturas largas con ayuda de las lecturas cortas. Esto mejora significativamente la calidad del ensamblado final.
  3. Ensamblaje final: generar el ensamblado completo usando los datos de larga lectura corregidos.

La elección entre Canu y Flye depende del tipo de datos, la capacidad de memoria disponible y el tiempo de cálculo requerido. Canu tiende a ser más preciso pero más lento, mientras que Flye es más rápido pero puede requerir más ajuste de parámetros.

Ejemplo práctico: uso de Canu con datos híbridos

Un flujo de trabajo típico con Canu para integrar datos híbridos sería:

canu -p my_genome -d assembly_dir \
  genomeSize=3.5g \
  -pacbio-raw reads_pacbio.fastq.gz \
  -nanopore-raw reads_ont.fastq.gz \
  -corrected-reads corrected_reads \
  -diploid \
  useGrid=false

Este comando inicia el ensamblado con datos de PacBio y ONT, y Canu utiliza internamente datos de lectura corta si se proporcionan mediante el parámetro -corrected-reads. Si no se proporcionan datos de lectura corta, Canu intentará corregir errores mediante alineamiento de lectura.

Ejemplo práctico: uso de Flye con datos híbridos

Para Flye, el uso de datos híbridos requiere un paso previo de corrección. Un flujo común es:

flye --nano-raw reads_ont.fastq.gz \
  --plasmid reads_plasmid.fastq.gz \
  --out-dir flye_output \
  --threads 16 \
  --min-asm-coverage 20

Después del ensamblado inicial, se puede utilizar minimap2 para alinear los datos de lectura corta al ensamblado, y luego medaka para corregir errores en el ensamblado utilizando las lecturas cortas como referencia. Este enfoque combina la velocidad de Flye con la precisión de datos de lectura corta.

Resumen

La integración de datos de secuenciación de larga lectura con Canu y Flye es una estrategia poderosa para generar ensamblados genómicos de alta calidad, combinando la longitud de lectura con la precisión de los datos de lectura corta. Canu ofrece un flujo de trabajo unificado y robusto para corrección y ensamblaje, mientras que Flye se destaca por su velocidad y eficiencia. Ambas herramientas pueden aprovechar datos híbridos para mejorar la continuidad y la exactitud del ensamblado, haciendo viable el estudio de genomas complejos, como los de organismos eucariotas y microbiomas.

Errores comunes y buenas prácticas

  • No proporcionar datos de lectura corta cuando se espera corrección: Si se usa Canu o Flye en modo híbrido, asegurarse de incluir datos de lectura corta en el flujo de trabajo.
  • Subestimar el tamaño del genoma: Un valor incorrecto de genomeSize en Canu puede llevar a ensamblados incompletos o con artefactos.
  • No filtrar datos previamente: Usar herramientas como fastp para eliminar contaminantes, bajos rendimiento y artefactos antes de el ensamblado.
  • Usar demasiados hilos: Flye y Canu son intensivos en memoria; ajustar el número de hilos según la capacidad del sistema.
  • Ignorar el análisis de calidad post-ensamblado: Verificar la calidad del ensamblado con herramientas como QUAST o Busco para evaluar contiguos (N50), cobertura y presencia de errores.
