Detección de dobletes en scRNA-seq

¿Qué son los dobletes en scRNA-seq y por qué son un problema?

En las experimentos de secuenciación de ARN a escala única (single-cell RNA sequencing, scRNA-seq), un doblé (doublet) se refiere a un archivo de secuenciación que proviene de la mezcla de material genético de dos células distintas durante el proceso de encapsulación. Esto ocurre cuando dos células se encierran en la misma cápsula (por ejemplo, en plataformas como 10x Genomics) y se secuencian como si fuera una sola célula.

Los dobletes introducen sesgos graves en el análisis bioinformático. Pueden generar perfiles de expresión que no corresponden a ninguna célula real, haciendo que se interpreten como nuevas poblaciones celulares o que se distorsionen las diferencias entre tipos celulares existentes. Esto puede llevar a conclusiones erróneas en estudios de heterogeneidad celular, diferenciación, o identificación de biomarcadores.

Herramientas de detección de dobletes

Existen varias herramientas especializadas para detectar dobletes en datos de scRNA-seq. Estas se basan en diferentes principios estadísticos y de modelado de datos, pero todas buscan identificar células con perfiles de expresión atípicos que no pueden explicarse como una única célula.

Scrublet

Scrublet es una herramienta popular basada en simulación de dobletes. Funciona generando un conjunto de perfiles de doblete sintéticos a partir de las células reales del conjunto de datos. Para cada célula, se calcula la distancia al perfil de doblete más cercano. Si esta distancia es significativamente menor que la esperada en una población de células únicas, se marca la célula como un doblete.

python -m scrublet --input data_filtered.h5 --output results_scrublet --n_jobs 4

DoubletFinder

DoubletFinder utiliza un enfoque basado en modelado de distribuciones de expresión. Se basa en la idea de que los dobletes tienden a tener una expresión de genes más alta y más dispersa que las células únicas. El algoritmo entrena un modelo de mezcla gaussiana para separar células únicas de posibles dobletes, utilizando características como el número total de genes expresados y la suma de expresión.

library(DoubletFinder)

# Cargar datos
obj <- Read10X(data.dir = "path/to/filtered/feature/barcodes/")

# Encontrar dobletes
obj <- DoubletFinder(obj, pct.doublets = 0.1)

scrublet en Python con Scanpy

Si estás usando Scanpy, puedes integrar Scrublet de manera nativa:

import scanpy as sc
import scrublet as scr

# Cargar datos
adata = sc.read_h5ad("data_filtered.h5ad")

# Crear objeto Scrublet
scrub = scr.Scrublet(adata.X, expected_doublet_rate=0.1)

# Detectar dobletes
doublet_scores, doublet_labels = scrub.scrub_doublets(
    n_neighbors=10, 
    min_dist=0.5, 
    n_pcs=30
)

# Añadir resultados al anndata
adata.obs["doublet_score"] = doublet_scores
adata.obs["doublet_label"] = doublet_labels

Prácticas recomendadas y errores comunes

Errores comunes

  • Ignorar la detección de dobletes: No realizar esta etapa puede llevar a conclusiones erróneas. Incluso un porcentaje bajo de dobletes (1-5%) puede afectar significativamente los resultados.
  • Usar una tasa de dobletes esperada demasiado alta: Si se establece un valor muy alto (por ejemplo, > 0.2), se pueden rechazar muchas células únicas. La tasa esperada debe basarse en el protocolo y en la calidad del experimento.
  • Confundir células con baja expresión con dobletes: Las células con baja cantidad de ARNm (por ejemplo, células muertas o en estado de estrés) pueden parecer dobletes si no se filtran previamente.

Buenas prácticas

  • Filtrar primero por calidad: Eliminar células con muy pocos genes expresados, alto contenido de genes mitocondriales, o baja suma total de expresión antes de detectar dobletes.
  • Validar con datos biológicos: Si se sabe que hay dos tipos celulares muy similares, un doblete entre ellos podría ser difícil de detectar, pero también menos problemático.
  • Usar múltiples herramientas: Comparar resultados entre Scrublet, DoubletFinder, y otros métodos puede aumentar la confianza en los resultados.
  • Visualizar resultados: Usar Scanpy para visualizar los dobletes en un UMAP o t-SNE, y verificar si están agrupados en regiones específicas del espacio de expresión.

Resumen

La detección de dobletes es una etapa crítica en el análisis de datos de scRNA-seq. Los dobletes no solo distorsionan las expresiones génicas, sino que también pueden crear falsas poblaciones celulares. Herramientas como Scrublet y DoubletFinder permiten identificar estos artefactos de manera robusta, especialmente cuando se usan con un filtro previo de calidad celular. La integración de estas herramientas en el flujo de trabajo con Scanpy o Seurat es esencial para obtener resultados biológicamente válidos y reproducibles.
