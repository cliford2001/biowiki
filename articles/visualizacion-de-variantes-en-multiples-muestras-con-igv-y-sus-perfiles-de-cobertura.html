Visualización de variantes en múltiples muestras con IGV y sus perfiles de cobertura

Introducción

La identificación y validación de variantes genéticas en múltiples muestras es un paso crítico en estudios de genética humana, cáncer y enfermedades hereditarias. Aunque herramientas bioinformáticas como GATK, FreeBayes o VarScan permiten la llamada de variantes de forma automatizada, la verificación visual de las llamadas es esencial para garantizar su fiabilidad. En este contexto, el Integrative Genomics Viewer (IGV) se ha consolidado como una herramienta indispensable para la exploración interactiva de datos de secuenciación de alto rendimiento (NGS).

En este artículo, exploraremos cómo utilizar IGV para visualizar variantes en múltiples muestras simultáneamente, prestando especial atención a los perfiles de cobertura, que ofrecen información crucial sobre la calidad de la llamada de variantes, la homocigosis, heterocigosis, y posibles artefactos técnicos. Aprenderemos a configurar sesiones múltiples, cargar archivos de variación (VCF) y archivos de cobertura (BAM), y a interpretar visualmente los datos para detectar errores comunes y aplicar buenas prácticas.

Preparación de datos y carga en IGV

Antes de abrir IGV, es necesario asegurarse de que los archivos de entrada estén correctamente formateados y indexados. Para visualizar variantes en múltiples muestras, necesitamos:

  • Archivos BAM o CRAM de cada muestra, indexados con samtools index o picard BuildBamIndex.
  • Un archivo VCF que contenga las variantes llamadas, preferiblemente en formato comprimido (.vcf.gz) y indexado (.tbi).

Una vez preparados, abrimos IGV (versión 2.10 o superior). Para cargar múltiples muestras, seleccionamos File > Load from File y elegimos los archivos BAM. Alternativamente, podemos usar el menú File > Load from URL si los archivos están accesibles públicamente. Después de cargar todos los archivos de muestra, añadimos el archivo VCF mediante File > Load from File o Load from URL.

Visualización de variantes y perfiles de cobertura

Una vez cargados los archivos, IGV muestra automáticamente los perfiles de cobertura para cada muestra en la región de interés. Estos perfiles se representan como gráficos de barras que indican el número de lecturas alineadas en cada posición del genoma. El alto nivel de cobertura en una región puede indicar una duplicación o un artefacto de enriquecimiento, mientras que una baja cobertura puede comprometer la fiabilidad de las llamadas de variantes.

Interpretación de variantes en múltiples muestras

Los puntos de variación en el archivo VCF aparecen como marcadores en el navegador. Para visualizar si una variante está presente en múltiples muestras, IGV muestra una fila para cada muestra en la ventana de cobertura. Las variantes homocigotas se muestran con lecturas de un solo tipo (por ejemplo, todos A o todos T), mientras que las heterocigotas muestran lecturas de ambos alelos.

Uso de la vista de múltiples muestras

Para facilitar la comparación, IGV permite crear sesiones múltiples mediante la herramienta Session > Save Session. Esto permite guardar el estado actual (regiones visibles, archivos cargados, marcadores de variación) y compartirlo con otros investigadores. También es útil para comparar variantes entre condiciones (por ejemplo, tumor vs. normal) o entre poblaciones.

Errores comunes y buenas prácticas

La visualización de variantes en IGV, aunque poderosa, no está exenta de errores. A continuación, se describen algunos escenarios comunes y recomendaciones para evitarlos:

Errores comunes

  • Cobertura inconsistente entre muestras: Una muestra con cobertura muy baja (<10x) puede llevar a falsos negativos en la llamada de variantes. Se recomienda verificar la cobertura media por muestra.
  • Artifacts de enriquecimiento: Si una región muestra cobertura anormalmente alta (más de 1000x), puede ser un artefacto de secuenciación o enriquecimiento. Revisar la calidad de las bibliotecas es clave.
  • Variantes que no se alinean correctamente: Algunas variantes, especialmente en regiones repetitivas, pueden mostrar alineaciones erráticas. Usar herramientas como gatk MarkDuplicates y gatk BaseRecalibrator antes de la llamada ayuda a reducir esto.

Buenas prácticas

  • Validar variantes en múltiples muestras: Si una variante está presente en todas las muestras, considerar si podría ser un error de alineación o contaminación.
  • Usar filtros de calidad: Añadir filtros VCF (como QUAL, DP, AF) antes de cargar en IGV para reducir el ruido.
  • Comparar con bases de datos: Usar herramientas como dbSNP o gnomAD para verificar si una variante es común o rara.

Resumen

La visualización de variantes en múltiples muestras con IGV es una práctica fundamental para la validación de resultados bioinformáticos. Al combinar la información de llamadas de variantes (VCF) con los perfiles de cobertura (BAM), los investigadores pueden detectar errores técnicos, confirmar la presencia de variantes biológicamente relevantes y tomar decisiones informadas sobre sus análisis. Las buenas prácticas, como la verificación de cobertura, el uso de filtros y la comparación entre muestras, son esenciales para garantizar la integridad de los resultados. IGV sigue siendo una herramienta esencial en el arsenal de cualquier bioinformático, especialmente en estudios de variantes en múltiples muestras.
